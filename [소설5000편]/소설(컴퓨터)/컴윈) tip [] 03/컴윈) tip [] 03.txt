[질문]
윈도우 98의 cap 파일에 들어 있는
파일들의 목록을 확인하는 방법이 없는가?
[답변]
Extract 명령을 여러 가지 스위치와 함께 사용하여
그 결과를 파일에 연결하면
윈도우 98 원본 캐비닛(Win98_02.cab로 시작)에 있는
모든 파일 목록을 볼 수 있다.
DOS 프롬프트에서 다음 명령을 입력한다.
extract /d /a win98_02.cab > List98.txt
참고로 cab은 캐비닛(cabinet)을 의미한다.
설치 과정에서 파일을 캐비닛에서 꺼내 온다는 의미에서 붙여진 것이다.

 제  목:[TIPS] 윈3.1과 윈98을 함께사용?!                            
 올린이:x배트맨x(손무홍  ) 98/12/13 10:57  읽음:88   추천:  0 관련자료 없음
 -----------------------------------------------------------------------------

[질문]
윈도우 98 설치 프로그램은 두 가지 옵션을 제공한다.
하나는 윈도우 98를 별도의 새 디렉토리에 설치하는 것이고,
다른 하나는 윈도우 3.x 디렉토리에 덮어서 설치하는 것이다.
그러나 윈도우 3.x 디렉토리에 설치하면
두 버전을 함께 사용할 수 없다.
윈도우 98를 별도의 새 디렉토리에 설치해도 문제는 있다.
윈도우 98가 윈도우 3.x에 미리 설정되어 있는 것을
그대로 계승하여 사용할 수 없기 때문이다.
또한 Win.ini와 System.ini에 있는 응용 프로그램의 설정 사항도
사용할 수 없다.
이 설정을 사용하려면 응용 프로그램을 모두 다시 설치해야 한다.
윈도우 3.1과 윈도우 98를 함께 사용할 수 있는 묘수는 없는가?
[답변]
둘 모두의 이점을 누리려면
윈도우 3.x 디렉토리를 다른 곳에 따로 복사해 두고
윈도우 98를 윈도우 3.x 디렉토리에 업그레이드 설치한다.
그러면 윈도우 3.x 설정 사항이 계승될 뿐 아니라
이중으로 시동할 수 있다.
이와 같은 설치 방법은 다음과 같다.
### 윈도우 3.x 복사 디렉토리에 윈도우 98 설치하기
* 단계 1
윈도우 3.x를 종료하고 C: 프롬프트로 되돌아간다. 
* 단계 2
C: 프롬프트에서 Xcopy C:\Windows\*.* C:\Win98\*.* /e를 입력한다.
이 명령은 윈도우 3.x 파일 전체를 복제한다.
위 명령에서 윈도우 3.x 파일이 있는 디렉토리 이름과
윈도우 98 파일을 설치할 디렉토리 이름을 자신의 환경에 맞게
바꾸어 주면 된다.
/e 스위치는 빈 디렉토리를 포함하여 모든 하위 디렉토리를 함께 복사한다.
프로그램이 TMP 디렉토리처럼
초기에는 사용되지 않는 디렉토리를 만들 때 이 스위치를 사용하면 편리하다.
새 디렉토리에 윈도우 98를 설치하면 대부분의 파일을 교체하지만
필요한 파일을 모두 설치하려면 20MB가 여분으로 더 필요하다.
설치 프로그램이 디스크 공간이 부족하다는 메시지를 표시할 때
윈도우 3.x 파일을 덮어씀으로써 줄어들 공간은 고려하지 않을 수도 있다.
그러므로 실제로는 충분한 공간이 있는데
이런 메시지가 나타나면 그냥 무시하고 진행해도 된다.
* 단계 3
C: 프롬프트에서 Xcopy C:\DOS\*.* C:\DOSSave\*.* /e를 입력한다.
이 명령은 DOS 파일을 다른 디렉토리에 별도로 저장한다.
윈도우 98를 윈도우 3.x가 있는 디렉토리에 설치할 때
윈도우 98는 긴 파일 이름처럼 윈도우 98의 기능과 호환되지 않는
이전 DOS 파일을 삭제한다.
그러나 이런 파일 중 나중에 이중 시동을 할 때 필요한 것이 있으므로
이와 같이 별도의 디렉토리에 복사해서 삭제되지 않도록 저장한다.
* 단계 4
C: 프롬프트에서 Edit C:\Autoexec.bat를 입력한다.
Autoexec.bat 파일에서 C:\Windows 레퍼런스를 C:\Win98로 바꾼다.
파일을 저장한 후 종료한다.
같은 방법으로 Config.sys도 변경한다. 
그 다음 Edit C:\Win98\Progman.ini를 입력하고
C:\Windows 레퍼런스를 C:\Win98로 바꾼다.
불행히도 일부 윈도우 응용 프로그램(Word 포함)은
자신의 ini 파일에서 C:\Windows를 참조한다.
이 응용 프로그램이 윈도우 98에서 제대로 작동하도록 하려면
C:\Win98 디렉토리에 있는 ini 파일을 하나하나 모두 편집해야 한다.
ini 파일을 한꺼번에 모두 편집하려면 다음과 같은 명령을 사용한다.
For %f in (C:\Win98\*.ini) Do Edit %f
이 명령은 ini 파일을 차례로 읽어 들인다.
각 파일을 저장한 후 종료한다.
그리고 컴퓨터를 재부팅한 다음 윈도우 3.x가
이름을 바꾼 디렉토리(C:\Win98)에서 정상적으로 시작되는지 확인한다.
C:\Windows를 참조하는 아이콘이 있으면 C:\Win98를 참조하도록 바꾼다.
* 단계 5
이제 윈도우 3.x에 있다.
파일의 실행 명령을 선택한 후 REGEDIT/v를 입력한다.
이때 나타나는 등록 정보 편집기에서
'C:\Windows' 문자열을 모두 찾아서 'C: \Win98'로 바꾼다.
편집기를 종료하고 윈도우 98 설치 프로그램을 시작한다.
현재 C:\Win98 디렉토리에 윈도우 3.x 복사본이 들어 있을 것이다.
이 디렉토리를 디폴트 위치로 하여 윈도우 98를 설치한다.
* 단계 6
윈도우 98 탐색기를 사용하여
예전의 DOS가 들어 있던 C:\DOS 디렉토리를 삭제한다.
그런 다음 C:\DOSSave 디렉토리를 C:\DOS로 이름을 바꾼다.
이제 이전 버전의 DOS 유틸리티들이 복원된 것이다.
만약 윈도우 98를 실행하고 있는 중이라면
새로운 DOS 7.0 유틸리티가 들어 있는
C:\Win98\Command 디렉토리의 명령을 사용한다.
그러나 윈도우 3.x로 부팅하면
DOS 5나 DOS 6 버전의 파일이 들어 있는 C:\DOS에서 유틸리티를 실행한다.
어떤 방법이든 윈도우 98는 자동으로 상황에 맞게
해당 Autoexec.bat 파일을 실행하여 적당한 path를 지정해 줄 수 있다. 
* 단계 7
윈도우 98로 부팅하면
윈도우 3.x용 이전 Config.sys와 Autoexec.bat 파일이
Config.dos와 Autoexec.dos로 각각 바뀐다.
이 파일을 텍스트 편집기로 열어서 C:\Win98를 참조하는 라인을
C:\Windows(또는 윈도우 3.x 파일이 들어 있는 디렉토리)로 바꿔 준다. 
* 단계 8
MS-DOS로 빠져나가 Msdos.sys의 읽기 전용, 숨김, 시스템 속성을 해제시킨다.
C:\Windows\Command 디렉토리에서 Attrib -r -h -s C:\Msdos.sys라고 입력하면 된다.

그런 다음 Msdos.sys를 텍스트 편집기로 연다.
[Options] 섹션에서 다음과 같은 라인을 추가하고 파일을 저장한다.
BootMulti=1
* 단계 9
컴퓨터를 다시 부팅해 'Starting Windows 98…'라는 메시지가 화면에 나타나면
F8을 눌러 부팅 프로세스를 가로채고 시동 옵션이 나타나게 한다.
이 옵션 중에서 Previous version of MS-DOS를 선택한다.
그러면 시스템이 윈도우 3.x로 부팅할 것이다.
F8을 누르지 않으면 윈도우 98로 자동 부팅된다.

 제  목:[TIPS] 윈98은 필요없지만 도스7.X는?                         
 올린이:x배트맨x(손무홍  ) 98/12/13 10:58  읽음:156  추천:  2 관련자료 없음

[질문]
윈도우 98는 필요없지만 도스 6.2의 상위 버전이 도스 7.0은 탐이난다.
이것만 사용할 수 있는 방법은 없는가?
[답변]
윈도우 98를 삭제해도 DOS 7.0으로 부팅할 수 있다.
그리고 DOS 7.0에서 윈도우 3.x를 실행할 수 있다.
그러나 만약 윈도우 98를 윈도우 3.x 디렉토리에 설치했다면
윈도우 3.x를 다시 설치해야 할 것이다.
이렇게 되면 윈도우 98가 한 일은
단순히 DOS를 업그레이드한 것에 지나지 않게 된다.
### 윈도우 98를 삭제하고 DOS 7.0으로 돌아가기
* 단계 1
컴퓨터를 켠다.
화면에 윈도우 98를 시작한다는 메시지가 나타나면 F8 키를 누른다.
Command prompt only를 선택한다.
* 단계 2
DOS 7.0 폴더를 새로 하나 만든 후 파일을 이 폴더로 복사한다.
DOS 7.0 명령 프롬프트에서 다음을 입력한다. 
c: 
cd\
md\DOS70
cd\DOS70
xcopy \Win98\Command\*.*
xcopy \Win98\Emm386.exe
xcopy \Win98\Smartdrv.exe
xcopy \Win98\Command.com
xcopy \Win98\Msd.exe 
xcopy \Win98\Setver.exe
\Win98는 윈도우 98 폴더이다.
* 단계 3
윈도우 98 파일을 삭제한다.
DOS 7.0 명령 프롬프트에서 다음을 입력한다.
path c:\DOS70
deltree c:\WIN98
* 단계 4
\Win98나 \Win98\Command 디렉토리 대신 DOS70 디렉토리를 참조하도록
Autoexec.bat와 Config.sys 파일을 편집한다.
디스크 캐싱을 위해 Autoexec.bat에 Smartdrv를 다시 추가할 수도 있다.
그리고 메모리를 관리하기 위해 Emm386.exe를 Config.sys에 추가할 수 있다.
* 단계 5
DOS 7.0 명령 프롬프트에서 시작하도록 Msdos.sys 파일을 편집한다.
DOS 7.0 명령 프롬프트에서 다음을 입력한다.
attrib -r -h -s Msdos.sys
edit Msdos.sys
Msdos.sys의 [options] 부분에서 다음 행을 추가한다.
BootGUI=0
BootWin=1
Msdos.sys 파일의 편집을 마치면 저장한 후
명령 프롬프트에서 다음 행을 입력한다.
attrib +r +h +s Msdos.sys
* 단계 6
일부 파일을 추가로 삭제할 수 있다.
DOS 7.0 명령 프롬프트에서 다음을 입력한다.
del Setuplog.* 
del Bootlog.* 
del Setlog.* 
del Sudhlog.*
del Netlog.* 
del System.1st

한글 윈도우 98에서 플로피 드라이브를 인식하지 못하거나

정확히 인식하지 못해서 발생하는 것이다.
따라서 이 기능을 비활성화(Disable)시켜 두면 해결된다.

『윈도우즈-윈도우 98 활용과 팁 (go WIN)』 4578번
 제  목:[TIPS] 명령Prompt와 바탕화면 오가기                         
 올린이:x배트맨x(손무홍  ) 98/12/13 11:03  읽음:82   추천:  0 관련자료 없음
 -----------------------------------------------------------------------------

[질문]
명령 프롬프트와 윈도우 98 바탕 화면을 자유자재로 왔다갔다 할 수 있는 방법은?
[답변]
루트 디렉토리에 Win.bat 파일을 만들어 둘 수 있는데
만약 이 파일이 아무 것도 없이 비어 있으면
윈도우 98가 명령 프롬프트로 부팅된다.
Msdos.sys에 BootGUI=0으로 지정한 것과 같은 상태가 된다.
윈도우 98 바탕 화면이 아닌 명령 프롬프트로 부팅한 경우
명령 프롬프트에 win을 입력하고 Enter를 누름으로써
윈도우 98 바탕 화면으로 들어갈 수 있다.
이런 방식으로 윈도우 98를 시작할 경우 윈도우 98를 종료하면
'이제 시스템 전원을 끄셔도 됩니다' 하는 메시지가 나타날 때
다시 DOS 7.0 명령 프롬프트로 되돌아갈 수 있다.
mode co80을 입력하고 Enter를 눌러 준다.

마우스 오른쪽 단추를 누르면 파일 보내기라는 항목이 있다.

특정 파일을 마우스 오른쪽 단추로 선택하고 파일 보내기의
플로피 디스크 드라이브를 선택하면 파일이 복사된다.
아주 편리한 기능이라는 생각이 드는데,
이 메뉴를 잘 활용할 수 있는 방법에 대해 알려달라.
[답변]
여러 단축 아이콘을 \windows\SendTo 폴더에 넣어 두면
마우스 오른쪽 단추 메뉴를 발전소로 탈바꿈시킬 수 있다.
파일 보내기란 메뉴 명령은 거의 대부분의 마우스 오른쪽 단추 메뉴에서 찾아볼 수 있
다.
실제로 보낸다는 의미가 아니라
파일을 이 응용 프로그램이나 폴더로 끌어다 놓는다는 의미로 해석하면 된다.
그러나 끌어서 놓기를 처리하는 방식은 해당 프로그램마다 다를 수 있다.
응용 프로그램의 동작 방법은 명령줄 매개변수를 수정하여 어느 정도 바꿀 수 있다.
파일 보내기 메뉴에 포함된 응용 프로그램은
마우스 오른쪽 단추 메뉴의 다른 명령과 비슷하게 보인다.
이 명령을 이용하면 파일을 백그라운드 압축하거나 인쇄, 보기,
특정 편집기로 열기 등의 다양한 작업이 가능하다.
탐색기를 사용하여 Windows 폴더를 확장시켜 보면 아래에 SendTo 폴더를 볼 수 있다.

파일 보내기 메뉴의 명령이 어떻게 작동하는지 보려면
프린터 폴더에서 프린터 아이콘을 하나 선택하여 SendTo 폴더로 끌어다
단축 아이콘을 만들어 두고,
폴더나 탐색기 창에서 파일을 마우스 오른쪽 단추로 누른 다음
파일 보내기 메뉴 항목에서 해당 프린터를 선택한다.
그 외에도 다양한 항목을 SendTo 폴더에 추가할 수 있다.
예를 들어 서버 컴퓨터 폴더의 단축 아이콘,
등록되지 않은 형식의 파일을 열 때 사용할 프로그램의 단축 아이콘,
아래한글과 같은 응용 프로그램의 단축 아이콘,
간략히 보기(\Windows\System\Viewer 폴더의 Quickview.exe)의 단축 아이콘 등을
SendTo 폴더에 추가하여 작업을 좀더 효율적으로 운영할 수 있다. 
파일 보내기를 사용할 때 윈도우 98는
파일을 마우스 왼쪽 단추를 사용하여 파일 보내기 목록에 들어 있는 응용 프로그램,
폴더 또는 기타 개체로 끌어다 놓은 것처럼 인식한다.
예를 들어 파일을 폴더로 보낸다고 가정해 보자.
만약 이 파일이 폴더와 같은 디스크 드라이브에 있다면 이 파일은 폴더로 이동될 것이

다른 디스크 드라이브라면 복사될 것이다.
또 다른 예로 파일을 아래한글로 보낸다면 아래한글이 실행되면서
해당 파일을 읽어들일 것이다. 사실, .Hwp 확장자를 가진 파일은
마우스로 두 번 누르면 자동으로 이러한 기능을 수행할 수 있기 때문에
별다른 잇점이 없지만

제어판에 대한 바로가기을 만드는 방법도 있지만 다음 방법이 더 효과적일 것이다.
제어판을 시작 메뉴의 메뉴 항목으로 만들고 폴더 창이 아닌
단순한 메뉴로 표시하는 방법이 있다.
그 방법은 다음과 같다.
### 제어판을 메뉴 항목으로 변환하기
* 단계 1
시작 단추를 마우스 오른쪽 단추로 누르고 열기를 선택한다.
제어판을 최상위 시작 메뉴에 올려 놓고 싶으면 여기서 멈추고,
그렇지 않다면 제어판을 포함할 메뉴 항목/폴더를 찾을 때까지 계속 폴더 창을 연다
* 단계 2
시작 메뉴의 폴더 창에서 빈 공간을 마우스 오른쪽 단추로 누르고
새로 만들기 -> 폴더를 누른다.
* 단계 3
'새 폴더'라는 이름이 반전 표시되어 있는 상태에서
새로운 이름을 다음과 같이 정확하게 입력한다.
제어판.{21EC2020-3AEA-1069-A2DD-08002B30309D}
* 단계 4
Enter를 누르면 '새 폴더' 아이콘이 제어판 아이콘으로 바뀔 것이다.
* 단계 5
이제 시작 단추를 눌러 보면
제어판 아이콘과 그 안에 속해 있던 모든 아이콘이 메뉴로 나타날 것이다.
