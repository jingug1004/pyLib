 제  목:[TIPS] 문제화된 CONFIG.SYS 핸들지정                         
 올린이:x배트맨x(손무홍  ) 98/12/13 04:02  읽음:176  추천:  0 관련자료 없음
 -----------------------------------------------------------------------------

문제없이 실행되던
DOS용 데이터베이스 프로그램이 갑자기 실행되지 않는다.
파일 핸들이 모자란다는 에러 메시지를 내면서 실행을 거부한다.
Config.sys 파일의 Files 행에
이 프로그램이 요구하는 만큼의 파일 핸들을 지정하고
시스템을 다시 시작해도 마찬가지다.
어찌된 영문인지 모르겠다.
[답변]
모든 프로그램은 실행할 때 파일을 불러 사용한다.
DOS에서는
프로그램에서 읽어 들이고 열어 놓은 다양한 파일을 추적하기 위해서
필요한 만큼의 메모리를 따로 지정해 줄 수 있다.
Config.sys 파일에서 FILES=30과 같은 명령문으로 처리한다.
이것은 프로그램에서 파일을 조작하기 위해 사용하는
이름을 보관할 메모리를 준비시켜 두는 것이다.
이런 이름을 파일 핸들이라고 부른다.
윈도우 DOS 세션에서 상당히 많은 파일을 열어 놓고 사용하는
DOS 프로그램을 시작하면 다음과 같은 오류 메시지를 만날 수 있다.
Insufficient File Handles, Increase Files in Config.sys
이 메시지 자체도 문제를 안고 있다.
Config.sys에서 FILES=을 아무리 늘려도
이 메시지는 계속 나타날 것이다.
정확하게 문제를 해결하자면 다음과 같은 메시지로 바뀌어야 할 것이다.
Add "PerVMFiles=15" to the [386Enh] section of SYSTEM.INI.
If 15 is not enough file handles, increase the number to 20.
Config.sys에서 지정하는 파일 핸들 숫자는
응용 프로그램에서 필요한 파일 핸들 수를 말한다.
System.ini의 PERVMFILES=은
윈도우에서 실행되는 가상 시스템(윈도우 DOS 세션)에서
동시에 열 수 있는 파일 핸들 수를 말한다.
System.ini에 PERVMFILES=이 지정되어 있지 않으면
각 DOS 세션에서 10개의 파일 핸들만 사용할 수 있도록
디폴트로 설정되기 때문에 일부 DOS 프로그램에선
파일 핸들 수가 부족한 현상이 발생하게 되는 것이다.
윈도우는 Config.sys에 30개의 파일 핸들을 지정할 것을 권장한다.
그러나 System.ini의 가상 시스템에 대한 파일 핸들 수는
이러한 오류 메시지가 나타났을 때에만 변경하는 것이 좋다.
각 파일 핸들은 작지만 메모리를 차지하기 때문이다.
DOS에서는 겨우 몇 바이트 정도밖에 소모되지 않는다.
Config.sys의 FILES= 라인에서 지정하는 핸들 숫자와
System.ini의 PERVMFILES=의 핸들 수는 모두 합쳐
255 이상을 넘을 수 없다.
물론 이렇게까지 많은 숫자를 사용하는 사람은 없을 것이다.

 올린이:x배트맨x(손무홍  ) 98/12/13 10:42  읽음:184  추천:  1 관련자료 없음
 -----------------------------------------------------------------------------

[질문]
아주 좋아하는 게임 프로그램이 하나 있는데
이 게임은 기본 메모리를 아주 많이 요구한다.
그런데 기본 메모리를 늘리는 온갖 방법을 동원해도
이 프로그램이 요구하는 만큼의 기본 메모리를 확보할 수가 없다.
기본 메모리를 좀더 늘릴 수 있는 방법을 알려 달라.
[답변]
기본 메모리를 엄청나게 필요로 하는 DOS 프로그램이지만
중첩 확장 메모리(EMS 메모리)는 필요하지 않다면
Emm386.exe를 Config.sys 파일에서 로드할 때
64K 페이지 프레임을 만들지 않도록 하여
좀더 많은 기본 메모리를 확보할 수 있다.
Emm386.exe를 사용한다면
Config.sys에서 다음과 같이 라인 끝에 frame=none을 추가하면 된다.
device=c:\windows\emm386.exe frame=none
이 매개변수는 Qemm386에서도 같이 쓸 수 있지만
다른 메모리 관리자에는 해당되지 않는다.
frame=none이란 매개변수는 특별한 환경에서 적용되지 않는 경우도 있다.
일반적으로 이 매개변수를 지정하면
Emm386 라인에 noems를 지정하는 것보다
좀더 많은 메모리를 확보할 수 있다.
그러나 frame=none이 현재 시스템에서 사용할 수 없는
매개변수라면(예를 들어 윈도우 98를 시작할 수 없는 문제가 발생)
noems 매개변수를 대신 사용한다.
Config.sys의 메모리 관리자에서 페이지 프레임을 삭제하게 되면
윈도우 98에서도 페이지 프레임을 만들지 못하도록 해야 할 것이다.
메모장을 사용하여 System.ini의 [386Enh] 섹션에 다음과 같은
라인을 추가하면 된다.
[386Enh]
NoEMMDriver=TRUE
Emm386이 64K 페이지 프레임을 만들지 못하도록 하면
64K만큼의 상위 메모리 블록(UMB)을
장치 제어기 및 TSR을 로드하는 데 사용할 수 있다.
지금과 같이 메모리를 구성하기 전에 UMB에 올라가지 못하던 제어기도
이젠 충분히 UMB에 로드될 수 있을 것이다.

 제  목:[TIPS] 화면배색에 없는 색상항목 변경                        
 올린이:x배트맨x(손무홍  ) 98/12/13 10:46  읽음:272  추천:  0 관련자료 없음
 -----------------------------------------------------------------------------

[질문]
화면 배색에서 바탕 화면에 나타나는 각 항목의 색상을 바꿀 수 있다.
그러나 바탕 화면에는 나타나지만 화면 배색에는 없는 항목이 있다.
이 항목의 색상도 바꿀 수 있는가?
[답변]
바꾸려는 항목이 사용할 수 없는 메뉴 항목의 색일 수 있다.
이 항목은 바탕 화면에서 볼 수 있지만 항목 필드에 나타나지는 않는다.
그러나 레지스트리에서 바꿀 수 있다.
이 색은 사용할 수 없는 메뉴 항목의 색뿐만 아니라
탐색기에서 플러스 기호를 둘러싼 사각형의 색과
왼쪽 창에서 점선으로 연결된 항목의 색을 바꿀 수 있다.
사용할 수 없는 항목의 표시법을 바꾸는 방법은 다음과 같다.
### 사용할 수 없는 메뉴 항목의 색상 바꾸기
* 단계 1
Windows 폴더에서 Regedit.exe를 두 번 누른다. 
* 단계 2
레지스트리 편집기 창에서 HKEY_USERS 옆의 플러스 기호를 누른다.
.Default -> Control Panel 옆의 플러스 기호를 차례로 누른다. 
만약 사용자 초기화 파일을 사용할 수 있으면

사무실 프린터에 문서를 인쇄할 수 있다.
이때 사무실 프린터는 공유되어 프린터 폴더에 들어가 있어야 한다.
첫 단계는 사무실 프린터를 공유하는 것이다.
사무실 프린터가 윈도우 98 컴퓨터에 있으면 시작 단추를 누르고
설정에서 프린터를 선택하여 공유한다.

공유 탭을 누르고 공유 옵션을 선택한다.
사무실 프린터는 집에 있는 컴퓨터의 프린터 폴더에서 보여야 한다.
집 컴퓨터의 프린터 폴더에서 프린터 추가 아이콘을 두 번 누른다.
프린터 추가 마법사에서 프린터가 네트워크 프린터라는 것을 지정하고,
\\server\Hp와 같은 UNC를 사용하여 프린터에 경로 이름을 지정한다.
(찾아보기 단추를 눌러 선택해도 된다.)
현재 사무실 프린터에 연결되어 있는 경우에는
프린터 제어기가 자동으로 그 컴퓨터에서 다운로드된다.
사무실 프린터가 공유되어
집에 있는 컴퓨터의 프린터 폴더에 나타나면 인쇄할 수 있다.
