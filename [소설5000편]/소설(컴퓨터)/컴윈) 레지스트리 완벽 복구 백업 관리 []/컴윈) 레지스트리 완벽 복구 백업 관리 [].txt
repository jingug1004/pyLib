０. 악~ 쓰바.. 돗되따

  윈도우를 쓰다보면 언젠가 한번쯤은 '레지스트리'를 날리게 되는
  비운의 참사를 맞이하게 될 수 있습니다.

  '마하핫~ 난 절때루 그런거 엄써!' 라고 자신 하시는 분 몇분 보이눈군여?

  하지만, windows 95 / 98 을 쓰면서 '안심' 이란 있을수 없습니다.
  필자는 어떠냐구요?  전 한번도 레지스트리 때문에 윈도우가 돗된 경우는
  없습니다.  본인은 직업상(?) 평균적으로 한달에 한번 윈도우를 '재설치'
  해야하기 때문에 부팅이 안되거나 에러란 있을수 없지여.

  그러나, 일반 유저(특히, 윈도우 설치CD를 가지고 있지 않은)은 윈도우가
  맛이가면 쉽게 재 설치할 수가 없지요.
  또, 자신만의 윈도우 환경을 일일이 새로 처음부터 만들어 주어야 한다는
  이중고를 격게 됩니다.

  이럴때를 대비하여 '시스템에서 중요한 레지스트리'를 백업시켜두는
  지혜가 필요합니다.


１. 시스템에서 중요한 화일들은 뭔가?

  레지스트리는 어디에 저장되는 것인가에 대해서 본 게시판에 이미 언급은
  했습니다만 다시 환기하는 의미에서 적어보겠습니다.
  우리의 친구(?) 빌(Bill)은 레지스트리를 획기적(?)으로 단지 2개의 파일
  USER.DAT 과 SYSTEM.DAT 로
  심플(?) 하게 나누어 저장하게 고안해 두었습니다.

  일반적으로 USET.DAT 은 윈도우의 사용자별 정보를 담고 있으며
  SYSTEM.DAT 은 시스템에 대한 총괄적 정보를 담고 있습니다.

  화일이 왜 두개로 나누어져 있느냐구요?
  일설에 의하면 "네트워크 관리의 편리성" 을 도모하기 위해서라고 합니다만,
  그리 신경쓸 필요는 없습니다.

  잠시 전 버전인 윈도우 95 에서의 백업 상황을 알아보면 windows 95는
  정상적으로 부팅하게 되면 위의 2개의 파일을 백업하게 되는데
  USER.DA0 와 SYSTEM.DA0 라는 확장자로 저장하게 됩니다.
  우리의 친구(?) 빌은 Windows 98 의 경우 이런 레지스트리 백업을 4개 씩이나
  작성하게 만들어 주었다고 합니다.

  친절한(?) 빌 덕분에 우리는 원한다면 적어도 리부팅 4회 전의 윈도우 환경
  으로 돌아 갈수 있습니다.

  윈도우가 제대로 부팅되지 않았을 경우에는 이 화일들이 어떻게 될까요?
  똑똑한 윈도우는 제대로 부팅되지 않았을 경우네는 바로 이전의 '백업'이
  더 정확한 것으로 판단(?)하고 새로 레지스트리 백업을 하지 않습니다.
  그래서 만약, 본 강좌를 따라하거나 다른 레지스트리를 건들이는 작업
  을 하다 윈도우를 정상적으로 부팅하지 못했을때 이 백업 화일을 복사하여
  바로 이전의 정상적인 윈도우 환경으로 돌아갈 수 있습니다.

  윈도우가 '멀티 유저'를 지원하기 때문에 백업화일들이 windows 폴더
  아래의 profiles 폴더에 있을 수도 있습니다.
  이건 여러명의 사용자 로그인이 존재한다는 뜻이겠지요.


２. 어떻게 복원 시킬까?

  기본적으로 복원 작업은 '도스 명령어'를 이해하고 있느냐에 관건이
  달려있다. '도스 세대' 에서 '윈도우 세대'로 넘어가신 분들은
  어느정도 '도스' 명령어에 대해 어설프지 않으리라 생각한다.

  그러나, 완벽한 GUI 윈도우 세대는 시커먼 화면에 '커셔'만 껌뻑껌뻑
  하고 있는 도스 환경과 그 명령어들을 이해하지 못할 수도 있습니다.

  그러나, 실망할 필요도 어려워 할 필요도 없습니다.
  그저.. 따라하기만 하면 됩니다.

  조건 : '윈도우로 부팅이 실패했다'

  1. Starting Windows 98 화면이 나왔을때 재빠르게 F8 키를 눌러라
     만약 이 작업에 매번 실패 했다면 Starting Windows 98 이라는
     메시지가 나오는 바로 이전 순간에 Ctrl 키를 지그시 누르고 있으면
     '부트 메니져' 화면이 뜨게 됩니다.

  2. '부트 메니져' 화면에서  "command prompt only" 메뉴를 선택
     도스 커맨드 명령 모드로 들어갑니다.

  3. 다음과 같이 따라하세요.

        C:\>path c:\windows\command
        C:\>cd windows
        C:\Windows\>attrib -h -s -r system.dat
        C:\Windows\>attrib -h -s -r user.dat
        C:\Windows\>del system.dat
        C:\Windows\>del user.dat
        C:\Windows\>cd sysbckup
        C:\Windows\Sysbckup>copy system.dat C:\windows
        C:\Windows\Sysbckup>copy user.dat C:\windows

　 ※ attrib 명령은 '화일의 속성'을 변경시키는 명령어 입니다.
      -h 는 히든 속성을 없애기
      -r 은 Read only 속성 없애기
      -s 는 시스템 속성 없애기 입니다.

   이 작업을 간단히 해설하면 윈도우 폴더로 이동한 후 레지스트리 파일과
   바로 이전에 백업된 레지스트리 파일의 속성을 일반 속성으로 해제하고
   백업된 파일을 원래 파일 위에 Overwrite (덮어 씌워 쓰는 것) 이다.
