０. Windows 98 을 빠르게?

  Windows 98을 빠르게 쓰려면 어떻게 해야합니까.
  Pentium II cpu 에 128M RAM 정도 있으면 됩니까? 비용이 만만치 않겠지요.
  여기, 현 시스템에서 조금이나마 빠른 윈도우즈 환경을 만끽할 수 있는
  방법 몇가지를 소개 합니다.


１. 윈도우즈 환경으로만 쓰겠다면?

  운영체계를 윈도우즈 만으로 정하겠다면 필요없는 도스용 메모리 관리 프로그램
  은 지워도 됩니다. 도스 시절에 컴퓨터를 맨 처음 기동시키면 읽어들이는 부팅
  화일에는 config.sys 와 autoexec.bat 화일이 있습니다.

  이 화일에서 메모리 관리 프로그램이나 디스크 캐쉬프로그램, 기타 램상주형
  프로그램 들을 등록해 두어 사용했었습니다. 하지만 Windows 95 나 98 에서는
  자체적으로 메모리 관리와 디스크 캐쉬 관리를 해주므로 이런 프로그램들이
  필요 없습니다. 이 프로그램들을 삭제해 주거나 주석처리(Rem) 해주면 조금이
  나마 속도 향상 효과가 나타납니다.

  필요없는 프로그램들은
  config.sys 에서 Himem.sys 와 Emm386.exe 와 country.sys, biling.sys 등이
  있고 Gscdrom.sys 화일은 도스상에서 시디롬을 쓰기위한 디바이스 인데 윈도우즈
  만 쓰겠다면 이 장치는 필요없습니다.

  autoexec.bat 에서 Files 나 buffer 항목, 디스크 캐쉬인 smartdrv.exe, cd롬
  확장장치인 mscdex.exe 등을 없앨수 있습니다.

  시작 버튼의 [실행] 에서 [sysedit]를 입력하여 실행한 후, config.sys를 선택
  하고 위에 열거된 항목 들을 삭제해 주시거나 아니면 나중을 위해서 rem 을
  넣어주어 주석처리 시켜주시면 됩니다.


２. 전 전용선 쓰는데요?

  Windows 98이 느려지는 결정적인 요인 중의 하나가 바로 액티브 데스크톱이다.
  화면을 편리하고 멋지게 꾸밀 수 있다는 장점도 있고, 채널을 통해 인터넷으로
  쉽게 정보를 받을 수 도 있지만, 전용선을 쓰지 않고 시스템의 CPU 나 RAM이
  넉넉치 않을경우에는 심각하게 속도를 떨어뜨린다.
  따라서 액티브 데스크톱은 꼭 필요한 경우에만 잠깐씩 사용하고 보통 때는
  꺼두는 것이 좋습니다.
  꺼두는 방법은 바탕화면에서 오른쪽 버튼을 눌러 [등록정보] -> [웹] 에서
  '액티브 데스크톱을 웹 페이지 형식으로 보기' 항목을 비워주면 액티브 데스크톱
  을 끌 수 있습니다.


３. 지겨운 디스크 조각모음

  디스크 조각 모음 작업은 시간이 상당히 오래 걸립니다.
  하드 디스크의 용량이 클 수록, 하드 디스크에 들어있는 데이터가 많을 수록
  더욱 그러합니다. 따라서 조금이라도 조각 모음 시간을 줄이려면, 조각 모음
  실행시 모든 윈도우즈 프로그램들을 모두 종료시키고 [디스크 조각모음] 만을
  실행시킵니다. 트레이에 등록되있는 프로그램과 현재 쓰고있는 프로그램까지
  모두 종료하면 됩니다.
  프로그램이 많이 실행되어 있을 수록 디스크 조각모음의 속도는 느려지기 때문
  입니다.
  또한 윈도우 98의 디스크 조각 모음은 윈도 95에 비해 훨씬 강력해졌습니다.
  가장 좋아진 점은 자주 사용하는 프로그램을 스스로 파악해 한군데로 모아주어
  실행 속도를 빠르게 한다는 것 입니다.
  디스크 조각 모음을 실행하고 [설정] 에서 '프로그램의 시작시간을 단축하기 위해
  파일 재정렬'에 체크해 주시면 이 기능을 사용할 수 있습니다.

  참고로, FAT32 로 변환시킨 드라이브에서 [디스크 조각 모음] 은 그렇지 않는
  FAT16 상태에서 보다 훨씬 더 느립니다. 심지어 10% 정도 조각을 모은 후에는
  더이상 진행하지 않는것 처럼 보이기도 합니다. [자세히] 를 눌러 조각모음 상황
  을 보면 계속 조각모음을 하고 있는 것이 보입니다. 인내심을 가지고 기다리세요.

０. 가상메모리 관리(Swap File)

  윈도우즈는 자체적으로 메모리를 관리합니다. 그러나 자체적으로
  관리하는 메모리로는 최적의 성능을 내기 어렵습니다. 때문에 가상메모리
  라는 방법으로 부족한 메모리를 매꾸게 됩니다. 이 가상메모리 역시
  윈도우즈가 자동으로 관리를 합니다만, 사용자가 더 넉넉히 설정해
  주어서 시스템을 향상 시킬 수도 있습니다.


１. 스왑 파일(Swap File)을 빠르게
 　
  스왑 파일 혹은 윈도우의 "가상 메모리(Virtual Memory)"라는 단어는
  실제 RAM을 다 사용했을 때 사용하는 하드 드라이브의 공간이나 파일을
  의미합니다.

  [제어판] -> [시스템] -> [성능] 탭을 선택하여 "가상 메모리" 버튼을
  누릅니다. 디폴트 값인 "Windows가 가상 메모리 설정을 관리"를 그대로
  둬도 별 무리는 없습니다만, 사용자가 직접 이것을 관리하고 싶다면
  "사용자가 직접 가상 메모리 설정을 관리" 버튼을 선택합니다다.
  스왑 파일은 윈도가 설치된 드라이버가 디폴트로 설정되어 있을 것입니다.

  만약 가장 빠른 드라이브나 파티션이 따로 있다면 그것으로 교체합니다.
  더 빠르고 작은 파티션이 있다면 그걸 선택하는게 좋습니다.
  물론 먼저 디스크 조각 모음을 실행해야 더 빨라지겠지요?


２. 가상 메모리를 가장 빠른 디스크와 파티션으로

  가장 빠른 드라이브나 파티션을 선택했으면 최소값과 최대값을 선택해야
  합니다. 가상 메모리는 현재 설치된 실제 RAM의 크기에 따라 달라집니다.

  대개 최소값과 최대값은 2.5배 정도로 설정하면 됩니다.
  즉, "실제 메모리 크기 ×2.5 " 의 값을 넣어주시면 됩니다.
  예로서, 16MB의 램을 갖고 있다면 최소값과 최대값 모두 40MB 정도로
  설정해 두면 됩니다.
  64MB의 램이라면 160MB 정도가 되겠지요?

  이렇게 변경된 가상 메모리 값은 시스템을 재시동해야 적용됩니다.


３. 64MB 이상이면 충분하지 않나?

  실제 램이 64MB 이상이라면 충분한 램 용량입니다.
  그래서 가상메모리를 160MB 나 잡아두어서 무슨 필요가 있는지 의아해
  하실 것 입니다. 윈도우즈에서 프로그램을 두개 정도만 띄우고 사용
  하신다면 필요가 없습니다만, 많이 띄우면 띄울수록 실제 메모리의
  리소스는 줄어들고 하드디스크에 스왑해서 만든 가상메모리를 이용합니다.
  프로그램을 실행할때마나 윈도우즈는 기존의 프로그램이 가상메모리를
  이용하도록 떠넘겨 버린후 새로 실행시킨 프로그램을 RAM 으로 읽어들
  입니다. 많은 프로그램을 사용할때마다 그 일부분이나 전체를 가상메모리
  로 떠넘기려면 스왑화일의 크기는 커져야 겠지요.

  물론, 스왑화일에서 작동하는 프로그램은 속도가 느려집니다.
