♡ 알아두면 편한 윈도우 98 설치 가이드 ♡


윈도우 98을 설치할때 무심코 지나쳐 버리기 쉬운 것들에 대해 알려드리고
설치시 도움이되는 정보와 '에러'에 대처하는 방법들에 대해 알려드리기 위해
올립니다. window 98을 제대로 설치하시는데 많은 도움이 되면 좋겠습니다.


1. 설치하기 전에 오류를 막기위해 알아두어야할 것들.
===================================================

시스템에서 실행 중인 모든 바이러스 백신 프로그램의 사용을 중단 해야 합니다. 설치
하는 동안 이들 백신 유틸리티가 계속 실행되면 시스템이 작동을 멈출 수도 있습니다
. 왜냐하면 WINDOWS를 설치하게 되면 '부트 레코드'를 재설정하게 되는데
백신 유틸리티들이 이것을 바이러스 프로그램에 의한 '공격'으로 오인하게 되어
설치 프로그램을 중단시키거나, 설치 중 '에러'가 나게하는 원인이 됩니다.
그 프로그램이 TSR(메모리 상주 프로그램)인 경우에는  Autoexec.bat 과
Config.sys 또는 Win.ini 파일에서 해당 프로그램 관련 명령어를 제거해 줍니다.

백신 유틸리티를 중단시키는 방법은 몇가지가 있습니다.

1) 현재 시스템 메모리상에 로드되어 있는 프로그램을 삭제시킵니다.
2) 윈도우즈의 시작프로그램에 등록되어 있는 백신 프로그램을 일시 삭제 시킨다.
3) AUTOEXEC.BAT 과 CONFIG.SYS 에 등록되어 있는 백신 프로그램 관련 명령어
   들을 제거 시키거나 REMARK 시켜둔다.
4) window 95 나 window 3.1 에 설치되어 있다면 간단하게 해당 백신 프로그램을
   자체 삭제프로그램을 가동시키거나 '프로그램 추가/삭제' 에서 제거 시킨다.

또한, 대부분의 586급 시스템은 마더보드의 BIOS(바이오스)상에서 바이러스 예방
기능이 내장되어 있습니다. 이 기능이 작동하게 되면 '바이러스 활동' 또는
'마스터 부트 레코드'가 변경되었다는 경고표시가 나타나게되고, WINDOW 98의 설치
가 중단될 수도 있습니다. 설치를 올바르게 진행시키려면 '마스터 부트 레코드'의
변경을 허용해야 합니다.

컴퓨터를 부팅할때 <DEL> 키를 계속 누르고 있으면, 'CMOS SETUP' 이라는 화면이
나옵니다. 대부분의 시스템이 쓰고 있는 AWARD BIOS 의 CMOS SETUP을 예로 들으면
두번째 항목에 BIOS FEAURES SETUP 이 나오고 그 안으로 들어가면
Virus Warning 이라는 항목이 나옵니다. 이 항목에서 엔터를 치시고
Page up / page down 키를 이용해서 'Disable' 로 바꿔주시면 됩니다.

하드 디스크에 에러가 있는지 의심스럽거나, lost cluster/chain 등의 문제를 확인
하거나 수정하려면 디스크 검사를 실행해 주시면 됩니다.
윈도우즈 95를 쓰고 계시다면 자체 프로그램을 이용해 주시면 되고
도스를 쓰고 계시다면 scandisk 나 checkdisk 를 쓰시면 됩니다.

windows 95에서 windows 98을 설치할 경우, 안정적인 업그래이드를 위해 실행중인
모든 프로그램을 종료해 주십시오.
예를들어 화면 보호기, 전원 관리 설정 프로그램, 백신 프로그램, 시스템 관리
프로그램 등이 있고 설치 프로그램의 올바른 실행에 문제가 될 수 있는 다른 모든 프
로그램들도 종료하시면 됩니다.
종료 방법중에 확실한 방법은 <Ctrl+Alt+Del>키를 누른 후 [작업 종료]를 선택하여
SYSTRAY 와 EXPLORER를 제외한 모든 실행 중인 작업을 종료해 주면 됩니다.

windows 98을 설치하기 이전에 미리 읽어보시면 도움이 되실 것입니다.


2. Windows 98 설치용량에 대한 일반적인 정보.
============================================

1) Windows 98 업그레이드 버전 설치시:
Windows 98 업그레이드 버전을 사용하는 경우, 설치 시 업그레이드 자격 검사
(예: window 3.1에서 windows 95로 업그래이드시 win.com 파일 존재여부 검사등)
에서 windows 95 에대한 필요한 검색하게 됩니다. 시스템에서 Windows 98 이전 
버전이 검색되지 않는 경우에는 업그레이드 자격을 부여할수 있는 이전 버전의
windows 제공하라는 메시지가 표시됩니다.

2) Windows 98 설치시 하드 디스크 용량 요구 메시지:
Windows 98의 설치에 필요한 디스크 공간은 여러 가지 조건에 따라 변경될 수
있기 때문에, windows 설치 시 SETUP 화면에 표시되는 요구공간은 일반적인
Windows 98 설치에 근거하여 측정된 대략적인 크기에 불과합니다.
하지만 원활한 설치를 위해 표시된 크기 이상의 하드디스크 용량이 필요합니다.

3) Windows 95 에서 windows 98 로 업그레이드시:
대략 165 MB의 빈 디스크 공간이 필요하나 설치시 선택한 시스템 구성 및 옵션에
따라 필요한 공간은 120-290 MB로 그 범위가 정해집니다.

4) FAT16 드라이브에 Windows 98 전체(Full install) 설치시:
225 MB의 빈 디스크 공간이 필요하나 설치 시 선택한 시스템 구성 및 옵션에 따라
필요한 공간은 165-300 MB로 그 범위가 정해집니다.

5) FAT32 드라이브에 Windows 98 전체(Full install) 설치시:
175 MB의 빈 디스크 공간이 필요하나 설치 시 선택한 시스템 구성 및 옵션에 따라
필요한 공간은 140-220 MB로 그 범위가 정해집니다.

6) C 드라이브 이외의 드라이브(예: D, E 등의 하드디스크)에 Windows 98을
  설치하는 경우:
설치하는 동안 작성되는 시스템 파일 및 로그 파일을 위해 C 드라이브에
임시 Setup 파일을 저장하기 위한 대략 25 MB의 사용가능한 하드디스크 공간이 
있어야 합니다.

7) Windows 98 업그래이드 설치중 Windows 95 백업시:
50 MB의 빈 디스크 공간이 필요하며 백업하려면 설치하는 동안 설치메뉴에서 
[시스템 파일 저장] 옵션을 선택해 주십시오. 시스템 파일을 저장하면 아래와 
같은 경우를 제외하고는 설치도중 Windows 98의 설치를 취소하고 이전에 사용하던
운영 체제로 복귀할 수 있습니다.

 ## 복귀할수 없는 경우:
 - 압축된 드라이브에 Windows를 설치한 경우
 - Set up에서 지정된 디렉토리가 아닌 새 디렉토리에 설치하거나 복원할 수 있는
   이전 버전의 운영체제가 설치되어 있지 않은 경우
 - MS-DOS의 버전이 5.0 보다 이하인 경우


Tips !!
#######
하나.
Windows 98 은 FAT32의 완벽한 지원을 위하여 리얼 모드(Real-mode) 및 보호 모드
(Protection-mode) 커널이 변경됨에 따라 Windows 98 의 시동 디스크는 Windows 95
와 호환되지 않으므로 주의하시기 바랍니다.
Windows 98 설치 시 FAT32를 사용하지 않는 경우 즉, 일반 FAT16을 쓰는 하드디스크
에라도 만약을 위해서 시동 디스크를 작성하기를 권합니다.
시동디스크 작성을 선택하시면 3.5인치 디스켓 한장이 필요하며 그 안의 내용은
포멧되어 지워져 다시 기록된다는점 또한 유의하세요.
설치시간이 2분-3분 정도 더 걸립니다.

두울.
Windows 98에서는 더 이상 Windows 3.x의 프로그램 관리자를 지원하지 않습니다.
프로그램 관리자(Progman.exe) 프로그램 자체는 여전히 제공되어 실행할 수도
있지만 실행해 보면 관리자 그룹에 아무것도 포함되어 있지 않습니다.
Windows 95에서 windows 98로 업그레이드하는 경우에 기존에 있던 *.grp 파일
역시 모두 삭제됩니다.
Windows 3.x에서 Windows 98업그레이드하는 경우에는, 호환성 유지 차원에서
*.grp 파일이 시스템에 그대로 보존 되며 프로그램 관리자의 일부 기능 또한
계속 유지됩니다. 
때문에, 프로그램 관리자를 사용하려면 Windows 98로 업그레이드하기 전에 필히
progman.ini와 *.grp 파일을 백업해두어야 합니다.
하지만, 구시대의 유물쯤으로 되버린 천덕꾸러기를 아직까지 사용하시는 분은
안계시겠지요.

3. MS-DOS MODE 에서 Windows 98 설치
===================================

이전 버전의 Windows에서 SETUP 실행 시 문제가 발생하거나, 새로운
하드 드라이브(포멧된) 에서 처음으로 설치를 실행하는 경우 또는 기존의 Windows
에서 다시 재설치 하고 싶은 경우에는 , MS-DOS 모드에서 Windows 98의 설치 프로
그램을 실행하는 것이 에러없이 설치하는 방법중 하나입니다.

MS-DOS 모드에서의 설치는 다른 방법의 설치에 비해 일반적으로 더 많은 시간이
소요되는 반면, 가장 안정적인 설치 방법이므로 다른 설치 방법에 문제가 발생할
경우나 안전하게 안정적으로 설치하고 싶을 경우 MS-DOS 모드에서의 설치하기를
권합니다.


아래에 열거된 키를 사용하여 시스템의 부트 메니져를 불러낼 수 있고 '부트메뉴'
에서 선택하여 MS-DOS 상태로 부팅할 수 있습니다.

1) Windows 98 에서:
시스템 부팅 시 <Ctrl>키를 누르고 있으면, Windows 98 부트 메뉴가 화면에 표시
됩니다. (Windows 95에서 사용되던 <F8>키도 사용할 수 있지만 Windows 98은
부팅 시 "Starting Windows 98" 이라는 프롬프트가 표시되지 않으므로 <F8> 키를
누르는 시간을 정확히 알 수 없기 때문입니다.)

2) Windows 95 에서:
"Starting Windows 95" 프롬프트가 화면에 표시되면 <F8>키를 누르십시오.
 화면에 Windows 95 부트 메뉴가 표시됩니다.

3) MS-DOS 6.x:
보통 그냥 놔두어도 별 문제가 없습니다만, 설치에 방해가 되는 '특정한' 프로그램
이 로드되어질 수도 있으므로 다음과 같이 "Starting MS-DOS" 프롬프트가 화면에
표시되면 <F8>키를 누르십시오.
로드되어질 프로그램 드라이버와 생략시킬 시스템 파일들을 수동으로 선택할 수
있습니다.



## 시디롬 드라이버를 띄우는 방법  ##

1) MS - DOS 를 사용중 일때 :
MS-DOS에서 CD를 사용하여 Windows 98을 설치하려면 리얼 모드 CD-ROM
드라이버가 로드되어야 합니다. 만약, 이미 작성해 놓은 Windows 98 시동
디스크가 있는 경우 시동 디스크에 포함된 CD-ROM 드라이버를 사용할
수 있습니다.  시동 디스크가 없다면 CD-ROM 하드웨어와 함께 제공된 설치
프로그램을 사용하여 CD-ROM 드라이버와 프로그램을 하드디스크에
설치 시켜주어야 합니다.

2) Windows 95를 사용중인 경우:
현재 Windows 95를 사용하는 경우에는 리얼 모드 CD-ROM 드라이버가
설치되어 있을 수도 있습니다. MS-DOS 모드로 시스템을 종료한 후
CD-ROM 드라이버를 액세스할 수 있는 경우 아래의 설명에 따라 주십시오.

1) 시스템을 다시 시작한 후 "Starting Windows 95" 프롬프트에서
   <F8>키를 누릅니다.
2) "Command Prompt Only"를 선택합니다.
3) C:\ 프롬프트에서 ' DosStart.bat'를 입력합니다.

이렇게 하면 CD-ROM 드라이브를 액세스할 수 있습니다.

CD-ROM 드라이브를 사용할 수 있게 되었으면 Windows 98 CD가 들어 있는 드라이브
로 전환하고 'SETUP.EXE'을 입력해 주십시오.
설치 프로그램이 시작되고 WINDOWS 98 설치가 진행 됩니다.

4. Windows 98 하드디스크에서 바로 설치하기
==========================================

⊙하드 디스크에서 Windows 98 설치 프로그램 실행:

모든 설치 파일을 하드 디스크에 복사하여 하드 디스크로부터
Windows 98을 설치하면 파일 복사 및 디스크 I/O 와 CD롬에 관련된
대부분의 문제를 해결할 수 있습니다. 이 설치 방법을 사용할 경우
CD-ROM 드라이버가 메모리에 상주되지 않음으로서 일부 컴퓨터에서 발생될 수도
있는 메모리 관련 문제가 발생할 위험성을 없앨 수 있습니다.

또한, 설치에 필요한 디스크 공간이외에 추가로 120 MB 이상의 디스크 공간
을 확보해야 합니다. Windows 95에서 업그레이드하는 경우 일반적으로 165 MB의
공간이 필요합니다.


(1) Windows 95에서 설치:

 1) 설치 파일을 저장할 임시 폴더를 그 드라이브에 작성합니다.

 2) 작성한 임시 폴더에 Windows 98 CD의 Win98 폴더 내용을 복사합니다.
    이때 Windows 98 의 하위 폴더는 복사할 필요가 없습니다.

 3) 시스템을 다시 시작합니다. "Starting Windows 95" 프롬프트에서
    <F8>키를 누른 다음 [Safe Mode Command Prompt Only]를 선택합니다.

 4) 이제, Windows 98 설치 파일이 들어 있는 임시 폴더로 이동하여
    'SETUP'을 입력하면 됩니다.


(2) MS-DOS에서 설치:

 1) CD-ROM 드라이브를 사용할 수 있는지 확인합니다. 자세한 내용은
    밑에 올라온 강좌의 내용을 참조해 주십시오.

 2) 설치 프로그램에 필요한 디스크 공간이외에 120 MB의 디스크 공간을
    추가로 확보합니다. Windows 95에서 업그레이드하는 경우에는
    일반적으로 165 MB의 공간이 필요합니다.

 3) 설치 파일을 저장하기 위하여 빈 디스크 공간이 많은 드라이브에
    임시 폴더를 작성합니다. 임시 폴더를 작성하려면 원하는 드라이브로
    이동한 후 예를 들어서 'MD Win98'를 입력하면 임시 폴더가 만들어집니다.

 4) 이제, Windows 98 CD-ROM 드라이브의 Win98 디렉터리로 이동합니다.

 5) 'Copy *.* C:\Win98' 또는 'Copy *.* D:\Win98'를 입력하여 Windows 98
    설치 파일을 이미 만들어 놓은 임시 폴더로 복사합니다.

 6) 모든 파일이 임시 디렉터리로 복사되면 시스템을 재부팅 시키고 여러가지
    프로그램이나 드라이버 파일의 로드를 생략하게 되는 "클린 부트(Clean boot)"
    를 수행합니다.

 7) 파일을 복사한 임시 폴더로 이동한 후 'SETUP'을 입력하여 설치 프로그램을
    실행합니다.


하드디스크 공간이 충분하다면 하드 디스크에서 바로 설치하는 방법을 권합니다.
설치 속도가 빠르다는 장점도 있지만, 처음 Windows 설치한 후에는 사용하는 
하드웨어에 대한 최신드라이버 업데이트도 자주 이루어지고, 또 각자에 맞는 '운영환
경'을 세팅하는 작업을 많이 하기때문에 'Windows 98 원본 CD'를 넣어라는
말이 자주 나옵니다.
(예, 키보드를 종류(1) 에서 종류(3)로 바꾸고자 할 경우)
처음 Windows 98 을 설치한후 자신에게 맞는 환경으로 완전히 세팅된 후에는
'임시로 만들어논 폴더'는 제거시켜도 상관없습니다.
그 후로는 'cd-rom'에 들어 있는 원본화일을 이용하시면 되죠.


!! Tip !!

윈도우 cd 에 보면 cab 이라는 파일이 있습니다.
이것은 압축 파일인데, 그 안에 내용을 보려면 Extract.exe 파일로 보던지
Cabinet viewer 라는 Power toy 를 설치하던지 했었어야 했습니다.
다 아시겠지만, 요즘엔 Winzip 7.0 버전에서 지원해주더군요.
Winzip 을 설치하신 후 그냥 클릭만 하세요. 안의 내용도 볼수 있고
화일도 풀어낼수 있습니다.

간혹, windows 98을 설치한 후 어떤 파일이 없다던가, 화일이 변경되었다던가
하는 메시지가 나올때는 원본 화일이 필요하죠. 
그때 이용하세요.

◎ 클린 부트로 (Clean booting) 실행시키자.
==========================================

특정 프로그램의 장치(device) 드라이버, 유틸리티, 또는 램 상주 백신
프로그램을 사용하는 경우, 설치가 올바르게 진행되지 않을 수 있습니다.
시스템 시동 시 클린 부트를 수행하면 설치 관련 문제를 대부분 해결할 수
있습니다. 그리고 windows를 설치하기전에 미리 CMOS 에 가셔서 부팅 드라이브
를 A:드라이브로 변경시켜 주시는것을 잊지 마세요.

클린 부트를 실행시키는 방법은 아래와 같습니다.

1) Windows 95 또는 MS-DOS 시동 디스크(플로피 디스크)로 부트.

  Windows 95 의 시동 디스크는 CD-ROM 드라이버를 시동디스크를 만들때 자동
  으로 넣어주지 않지만 하드 디스크에 들어 있는 CD-ROM 드라이버 파일을
  사용자가 작성된 시동디스크의 여유 용량에 복사한 경우에 사용될 수 있습니다.
  또 Mdir 같은 작은 유틸리티도 넣어두면 설치할때 정말 편리합니다.
  Ms-DOS 의 시동 디스크도 역시 시동 디스크를 만들때 컴퓨터가 자동으로
  CD-ROM 드라이버를 넣어주지 않지만, 하드 디스크에 들어있는 자신의 CD-ROM
  드라이버 파일을 사용자가 시동 디스크의 여유 용량에 복사하면 사용할수
  있습니다. 이때, 간단한 화일관리 유틸리티(ex: Mdir) 를 넣어두는것도
  괜찮겠지요. 또, 메모리 관리자 (ex: himem.sys , emm386.exe , memmaker.exe)
  와 디스크 관리자 (ex: scandisk.exe , chkdsk.exe) 등을 넣어주시는 것도
  좋습니다.

2)  Windows 95 Safe Mode Command Prompt Only:

  시스템을 시작하고 "Starting Windows 95" 프롬프트에서 <F8>키를
  누릅니다.  [Safe Mode Command Prompt Only]를 선택합니다.
  Safe Mode Command Prompt Only 옵션은 리얼모드 CD-ROM 드라이버를
  지원하지 않습니다. 하드 디스크에 설치 파일을 복사해서 설치하는 경우에만
  이 옵션의 사용을 권장합니다.

3) 메모리를 확보하기 위한 Windows 95 / MS-DOS 클린 부트

  Config.sys 파일을 아래와 같이 수정하여 사용 가능한 메모리 영역을
  확보할 수 있습니다. 각 시동 디스크의 Config.sys 파일 또한 아래와
  같이 수정 할 수 있습니다.
  아래에 열거된 항목들은 충분한 메모리 확보를 위해 반드시 로드되어야 하는
  드라이버입니다.

  Device=Himem.sys
  Device=EMM386.exe noems
  Dos=high,umb

  만약 디스크 공간 늘림을 사용하는 분들은 아래의 항을 추가하세요
  Device=drvspace.sys /move

Config.sys과 Autoexec.bat 편집하는 방법
=======================================

시스템의 Config.sysdhk Autoexec.bat 파일은 컴퓨터가 기동할 때 로드될
프로그램과 장치를 지정해 주는 역할을 합니다.
예를 들면 Autoexec.bat 파일에서는 바이러스 백신 프로그램을 시스템에
자동으로 로드할 수 있도록 하는 등의 작업을 수행합니다.
또, config.sys 파일에서는 프로그램의 장치(디바이스(device))를 지정,
시스템에 로드 시킵니다.

Windows 98 설치 프로그램은 일부의 프로그램 또는 장치와 호환되지
않습니다. 이런 프로그램 또는 장치를 제거하거나 사용하지 않도록 설정하려면
Config.sys 또는 Autoexec.bat 파일을 편집해야만 하는 것입니다.


◎ Config.sys과 Autoexec.bat 파일을 수정해 보자!

(1) Windows 3.1 / 3.11 에서

  1. Windows 3.1이나 3.11을 사용하는 경우, [파일]을 클릭하고 [실행]을
     클릭한 후 Sysedit을 입력하고 <Enter>키를 누릅니다.

  2. Config.sys 또는 Autoexec.bat 파일 내용이 나타난 윈도우 박스에서
     사용하지 않도록 설정하려는 프로그램이나 장치가 들어있는 줄의 시작
     부분에 REM을 입력하여 주석으로 처리합니다.
     (Rem - 주석처리 라고도 하며 주석으로 된 부분은 실행되지 않습니다.)

  3.  변경 사항을 꼭! 저장하고 시스템을 재시작합니다.

(2) Windows 95 / OSR2 / 98 에서

  1. Windows 95 등 에서는 [시작] 버튼을 클릭하고 [실행]을 클릭한 다음
     Sysedit를 입력하고 <Enter>키를 누릅니다.

  2. 2. Config.sys 또는 Autoexec.bat 파일 내용이 나타난 윈도우 박스에서
     사용하지 않도록 설정하려는 프로그램이나 장치가 들어있는 줄의 시작
     부분에 REM을 입력하여 주석으로 처리합니다.
     (Rem - 주석처리 라고도 하며 주석으로 된 부분은 실행되지 않습니다.)

  3.  변경 사항을 꼭! 저장하고 시스템을 재시작합니다.

  4. 2번 방법 이외에 <시스템 정보> 에서 '도구'를 사용하여 간편하게 내용
     을 변경시키는 방법도 있습니다.
     이 부분은 차후 기회가 닿는데로 글을 써 보겠습니다.

(3) MS-DOS 에서

  1. DOS 디렉토리에 있는 EDIT.EXE 를 실행시킨다.

  2. 'FILE' 탭에 'LOAD'를 눌러 메뉴를 불러 들인다.

  3. AUTOEXEC.BAT 이나 CONFIG.SYS 를 찾아준다. <TAB> 키를 눌러 프롬프트
     를 이동시킨 후 루트(ROOT)디렉토리에서 찾아주면 된다.

  4. 변경 후 'FILE' 탭에 'SAVE' 를 눌러 저장시키고 빠져나온다.

◎ 설치 중 하드 디스크 문제의 검색 및 수정
===========================================

ⅰ. Windows 98을 설치하는 동안 문제 검색

Windows 98을 설치하는 동안 실행되는 디스크 검사 프로그램은 단지
오류를 확인하기만 하며 수정하지는 않습니다. 만약, 문제가 검색되면
설치 프로그램은 문제가 수정될 때까지 설치를 진행하지 않습니다.

문제를 수정하려면 설치를 종료한 후 Windows 95나 MS-DOS에서 해당버젼의
디스크 검사 프로그램 (WINDOWS 95 의 '디스크 검사' , DOS 의 SCANDISK.EXE)
으로 디스크 검사를 실행해야 합니다.

그러면, 디스크 검사를 사용하여 문제를 해결하는 방법을 설명하겠습니다.

ⅱ. 하드 디스크 문제 해결:

설치하는 중 하드 디스크에 문제가 발견되어 이를 수정하기 위해서 디스크
검사 프로그램을 실행해야 한다는 메시지가 표시되면 다음의 단계를 참조하여
문제를 해결 하면 됩니다.

1. MS-DOS나 Windows 3.1 버전에서 Windows 98을 설치하는 경우

 1) Windows 3.1 을 종료합니다. <- DOS 상태로 빠져나감.

 2) 설치 디스크가 플로피 디스크인 경우, 플로피 드라이브에 설치
    디스크 1번을 넣은 후 명령 프롬프트에 아래와 같이 입력합니다.

        a:scandisk.exe /all

    여기에서 "a:"는 Windows 설치 디스크가 들어 있는 드라이브입니다.

 3) CD(원본)로 설치하는 경우, CD를 드라이브에 넣은 후 아래와 같이 입력합니다.

        d:\win98\scandisk.exe /all

    여기에서 "d:"는 CD가 들어 있는 드라이브입니다.

 4) 화면의 지시에 따라 디스크 검사 프로그램이 검색한 문제를 수정합니다.

 5) Windows를 시작한 다음 설치 프로그램을 다시 실행하거나
    DOS 상태에서 설치 프로그램을 다시 실행합니다.

2. 기존에 있던 Windows 98에 덮어 씌우거나 Windows 95에서 Windows 98을
   설치하는 경우

 1) 설치를 중단합니다.

 2) [시작] 메뉴에서 [프로그램] -> [보조프로그램] -> [시스템 도구]의
    순서로 찾아간 후 [디스크 검사]를 선택합니다.

 3) 하드 디스크 및 모든 호스트 드라이브에 오류가 있는지 검사하여
    검색된 문제를 수정합니다. 모든 드라이브에서 표면 검사의 수행을
    해야만 합니다. 표면 검사의 수행을 완료하지 않으면 수정되지 않은
    오류에 의해 설치가 중단될 수도 있습니다.

    * 물리적인 디스크 표면 검사는 생략해도 좋습니다.

ⅲ. 디스크 검사 실행 후 문제 해결이 안되는 경우.

아래와 같은 경우에는 디스크 검사 프로그램이 문제를 해결할 수 없거나
오히려 문제의 원인이 되는 경우가 있을 수도 있습니다.

1. 디스크 공간 늘림 3을 사용하는 경우:
   디스크 공간 늘림 3이 시스템에 설치되어 기본 메모리가 부족한
   경우에 문제가 발생할 수 있으며, 메모리를 확보하려면 아래와 같이 행합니다.

 1) MS-DOS 6.x를 사용하는 경우에는 Memmaker.exe를 실행하여
    디스크 검사를 완료하는데 충분한 메모리를 확보할 수 있습니다.
 2) 메모리 확보를 위해 클린 부트를 수행하는 방법에 대하여는
    본 게시판의 게시물 중 "클린 부트를 수행하는 방법"를 참조해 주십시오.



## TIP! ##

설치 프로그램이 디스크 검사를 실행하는 동안 메모리가 충분하지
않거나 다른 문제가 발생하면 /IS 옵션으로 설치 프로그램을 실행하여
디스크 검사를 실행하지 않고 생략할 수 있습니다. 아래와 같이 입력합니다.

        setup /is

참고: 설치 시 디스크 검사를 실행하지 않고 생략하는 것은 그리 권장하지
않습니다. 검사를 생략한 하드 디스크에 문제가 남아 있는 경우에는
Windows 98의 설치가 올바르게 완료되지 않거나 진행 시 문제가 발생할
가능성이 있습니다.

◎ Windows 98 을 제거해야 할때
==============================


１. 시스템 파일 저장이란 ?

Windows 98 설치 프로그램은 설치한 Windows 98을 제거할 수 있도록
Windows 98 설치시 첫부분 화면에 이전 버전의 Windows를 백업하는 옵션을
제공합니다. 이 옵션을 사용하려면 설치 중 표시되는 [시스템 파일 저장]
옵션을 선택해야 합니다.
옵션을 선택하면 Windows 98 제거할 경우에 사용할 수 있도록 아래와 파일이
작성되며 이 파일들은 Hidden 속성을 가집니다.

  * Winundo.dat
  * Winundo.ini
  * Winlfn.ini

만약 위 파일들을 삭제되면 Windows 98을 제거할 수 없습니다.
또한, 아래와 같은 경우에는 설치시 [시스템 파일 저장] 옵션을 설정할 수
없습니다.

  # Windows가 압축된 드라이브에 설치된 경우

  # 새 디렉터리에 설치하거나 복원할 수 있는 이전 버전의 운영체제가
    설치되어 있지 않은 경우

  # MS-DOS의 버전이 5.0 보다 이하인 경우

참고적으로 Windows 98을 제거하는 데 필요한 파일은 반드시 하드 드라이브에
저장해두어야 합니다. 빈 공간이 있는 드라이브가 두 개 이상일 경우에는
제거 정보를 저장할 드라이브를 사용자가 선택할 수있습니다.

하지만, Windows 98 설치시 시스템 파일을 저장하였더라도 아래와 같은
경우에는 저장한 정보가 삭제되어 Windows 98의 제거를 수행할
수 없게 됩니다.

  # Windows 98 설치 후 하드 디스크를 FAT32로 변환한 경우
  # Windows 98 설치 후 하드 디스크를 디스크 공간 늘림으로 압축한 경우

물론, 위에 열거된 유틸리티는 변환이나 압축을 수행하기 전에 '시스템 파일
저장 제거 정보' 의 손실을 미리 알려줍니다.


２. Windows 98 제거을 제거해 보자.

⑴ Windows 98을 제거하고 시스템을 기존 MS-DOS나 Windows 3.1 또는
   Windows 95로 되돌아 가는 방법.

 1) [시작] 메뉴의 [설정]에서 [제어판]을 선택합니다.

 2) [프로그램 추가/제거]를 실행합니다.

 3) [설치/제거] 탭에서 [Windows 98 제거]를 선택한 후 [추가/제거]를
    누릅니다.

만약, Windows 98 으로 부팅에 문제가 있는 경우 시동 디스크를 사용하여
시스템을 시작한 후 시동 디스크에서 Windows 98의 UNINSTAL을 실행할 수
있습니다.


⑵ 시동 디스크에서의 Windows 98 Uninstall 방법

 1) Windows 98을 종료 합니다.

 2) 시스템을 재시작하고 "Starting Windows 98" 메시지가 표시될 때
    <F8>키를 눌러 [Command Prompt Only]를 선택하여 주십시요.

 3) Command 프롬프트에서 UNINSTAL을 실행해 주시면 됩니다.


⑶ 설치 프로그램이 완전히 종료되지 못한 경우에 Unintall 방법

  기존의 MS-DOS와 Windows 3.x 또는 Windows 95를 복원하려면

 - 하드 디스크의 C:\Windows\Command 디렉터리
 - 시동 디스크

   에서 UNINSTAL을 실행해 주십시오.

※ C 드라이브 이외의 사용자가 지정한 드라이브에 [제거 정보 파일]을
   저장한 경우에는 /w 옵션을 사용하여 파일이 있는 드라이브를 지정할
   수 있습니다.

        uninstal /w d:

  위의 명령 중 d:는 사용자가 따로 지정한 [이전 시스템 파일]이 들어 있는
  드라이브입니다.

３. Windows 98 사용하는 중 디스크 용량 확보를 위해 [이전 시스템 파일]
    을 삭제하는 방법.

  1) [시작] 메뉴의 [설정]에서 [제어판]을 선택합니다.

  2) [프로그램 추가/제거]를 실행합니다.

  3) [설치/제거] 탭에서 [이전의 Windows 3.x 및 MS-DOS 시스템 파일] 또는
     [Windows 95 시스템 파일 제거(제거 정보)]를 선택한 후 [제거]를
     클릭합니다.

 제거 정보를 삭제하면 당연히 Windows 95를 포함한 이전버전의 윈도우로
 되돌아 갈수 없습니다.

◎ 다른 OS 에서의 Windows 98 설치 방법


１. Windows NT 시스템에서 Windows 98 설치

원칙적으로 Windows NT에 Windows 98를 설치할 수는 없지만 하나의
시스템에 공존하도록 설치할 수는 있습니다.
호환성 문제가 발생할 수 있으므로 반드시!! 별도의 하드 디스크 또는
별도의 분할 영역에 개별적!! 으로 설치해야 합니다.

Windows NT 가 이미 설치되어 있는 경우 Windows 98을 설치하면
Windows NT 부트 메니져 메뉴에 Windows 98이 추가되어 사용자가
Windows 98 또는 Windows NT 중 사용할 운영 체제를 선택할 수 있도록
해 줍니다.

만약, 설치상에 문제가 발생하여 Windows NT로 시동할 수 없게 되면
Windows NT의 '복원 디스크' 를 사용하여 시스템을 부팅한 후 수정 옵션을
선택하여 Windows NT 부트 파일을 복원할 수 있습니다.

그럼에도 불구하고 Windows NT 시스템이 설치되어 있는 드라이브에
Windows 98을 설치하면 아래와 같은 오류가 발생한다고 합니다.

"시스템의 하드 디스크가 64K 클러스터의 FAT 분할 영역을 포함한
 것으로 검색되었습니다. 이와 같은 클러스터 크기를 포함한
 디스크에서는 디스크 검사 프로그램이 작동하지 않습니다. 설치를
 계속 진행할 수 없습니다. 설치를 완료하려면 반드시 하드 디스크를
 다시 분할하고 클러스터의 크기가 32K 이하인 FAT 파일 시스템으로
 분할 영역을 포맷한 후 설치 프로그램을 다시 실행해야 합니다."

" /is " 옵션을 사용하여 설치 프로그램을 실행하면 디스크 검사를
 실행하지 않고 생략하므로 오류의 발생을 막을 수 있습니다.

 예) Setup /is

 그렇다고는 한다지만, 설마 정말 설치하겠다고 맘먹으신분 없죠?


２. OS/2 가 설치된 시스템에서 Windows 98 설치

OS/2에서는 설치 프로그램이 아예 실행되지 않습니다.
MS-DOS로 시스템을 시작하여 MS-DOS 프롬프트에서 설치 프로그램을
실행해야 합니다.

참고적으로 HPFS 분할 영역을 사용하는 OS/2에서 Windows 98로 업그레이드를
하려면 설치 시 OS/2 1번 정본 디스크가 필요합니다.

플로피 디스크로 강제로 MS-DOS를 시작하여 Windows 98을 설치한
경우에는 더 이상 OS/2를 사용할 수 없습니다.
Windows 98 설치 프로그램을 실행하기 전에 OS/2가 사용하는 Autoexec.bat과
Config.sys 파일을 삭제해야 합니다.

※ Windows 98을 설치 한 다음 시스템에서 OS/2를 제거하려면 아래의
   절차를 실행해 주십시오.

 1) 계속 보관할 데이터 파일을 플로피 디스크 또는 이동형 저장장치
    백업 장치, 네트워크 드라이브등을 통해 백업합니다.

 2) OS/2 디렉터리와 하위 디렉터리에 있는 모든 파일을 삭제한 후
    OS/2 디렉터리를 삭제합니다.

 3) Hidden 속성을 가진 화일들이 남아 있을 수도 있습니다.
    [내 컴퓨터] 또는 [Windows 탐색기]의 [보기] 메뉴에서 [옵션]을
   누른 후 [모든 파일 표시]를 선택하여 아래의 OS/2 파일을 삭제하면 됩니다.

        Ea data.sf
        OS2ldr.msg
        OS2krnl
        OS2boot
        Wp data.sf

◎ 설치시 오류 해결 방법


1. "이전 버전 Setupx.dll" 메시지:

이전 버전의 SetupX.dll이나 NetDi.dll을 사용하는 중입니다.
시스템에 열려 있는 응용 프로그램이 이 문제의 원인일 수 있습니다.
기존의 파일을 대체한 다음 자동으로 시스템을 재시작 하려면 [확인]을
누르십시오. 기존의 SetupX.dll이나 NetDi.dll을 유지하려면 [취소]를
눌러 설치를 종료한 후 열려 있는 응용 프로그램을 닫은 후 설치
프로그램을 다시 실행해 주십시오.

이 오류를 원인이 될 수 있는 일부의 프로그램이 있습니다.

* Diamond의 "In Control Tools"
* Ensonics AudioPCI 사운드 카드 드라이버
* Aztech 사운드 카드 드라이버
* Hayes Modem 구성 유틸리티
* 일부 Windows 인쇄 시스템 유틸리티
  (Canon BJ 시리즈)

시스템을 다시 시작한 후에도 Windows 98 설치 과정에서 이 오류가
지속되면 MS-DOS 프롬프트에서 설치 프로그램을 실행하면 됩니다.

2. 메시지 SU0018

"시스템의 시동 드라이브에 파일을 작성할 수 없으므로 Windows 98을
설치할 수 없습니다. 시동 드라이브의 루트 디렉터리에 너무 많은
파일이 존재하거나 시동 드라이브 문자가 재지정되었을 수 있습니다. "

드라이브의 루트 폴더는 최대 512 개의 항목(파일이나 폴더)을 포함할
수 있습니다. 이 메시지는 시스템의 루트 폴더에 너무 많은 디렉터리
항목이 존재하여 Windows 98을 설치하는 데 필요한 파일을 작성할 수
없음을 의미합니다. 드라이브의 루트 폴더에서 일부의 파일을 이동
또는 삭제한 후 설치 프로그램을 다시 실행해 주십시오.

3. "회복할 수 없는 설치 오류" 메시지

" 회복할 수 없는 설치 오류. 이 시스템 구성에서는 설치를 계속할
수 없습니다. [확인]을 클릭해서 설치를 끝내십시오."

이 오류는 여러 가지 상황에서 발생할 수 있습니다.
의심해볼만 한 것은, 백신 프로그램과 램 상주 프로그램 충돌 등이며
해결 방법으로는 MS-DOS에서 Windows 98 설치를 들수 있습니다.

4. 긴 파일 이름 오류 메시지

"이 디렉터리에 설치된 Long File Names 응용 프로그램이
검색되었습니다. 설치를 계속할 수 없습니다."

이 메시지가 나오면 설치를 중단한 후, 하드 디스크 상의 Long File Names을
제거 프로그램을 사용하여 제거하면 됩니다.
자세한 내용은 "뷰 소프트웨어"를 참조해 주십시오.

5. 메모리 부족 메시지

메모리 부족 오류 메시지가 표시되면 모든 TSR 프로그램을 주석 처리
(Rem) 하고 장치 드라이버를 상위 메모리 영역으로 로드하여 기본 메모리를
늘릴 수 있습니다.

6. 디스크 공간 부족 메시지

아래와 같은 방법으로 디스크 공간을 확보할 수 있습니다.

 1) 휴지통을 마우스 오른쪽 단추로 클릭한 다음 [휴지통 비우기]를
    선택합니다.

 2) 인터넷 브라우저 캐시 폴더의 내용을 삭제합니다.
    (익스플로러 & 네비게이터 또는 E-mail 프로그램 캐쉬 등)

 3) 확장명이 .bak과 .tmp인 파일을 삭제합니다.

 4) 사용하지 않거나 중요하지 않다고 판단되는 프로그램 폴더를 삭제합니다.
    (물론, 필요한 데이터는 미리 백업하십시오.)

 5) Windows 98과 MS-DOS를 모두 사용하도록 시스템을 구성하지 않은 경우
    즉, Windows 98로만 사용하고자 할 경우 MS-DOS 폴더를 삭제합니다.
    (만일을 위하여 먼저 부팅 가능한 디스크에 CD-ROM 드라이브를 인식할
     수 있는 프로그램과 기타 지원 프로그램을 미리 복사해 둡니다.)

 6) 기존의 Windows 95의 숨김 속성을 가진 파일 Winundo.dat을 삭제합니다.

 7) Windows 3.1과 Windows 98을 모두 사용하도록 시스템을 만들어 놓지
    않은 경우에는, Windows 3.1 폴더를 과감히 삭제합니다.

7. "임시 디렉터리에 쓸 수 없습니다."

이 메시지는 임시 디렉터리의 디스크 공간이 부족할 때 나타날 수
있습니다. 다른 드라이브에 사용할 수 있는 공간이 있으면 아래의 옵션
명령을 사용하여 임시 디렉터리 위치를 변경합니다.

        Setup /T:<drive letter>:\TEMP

  예) setup /T:d:\temp

다른 드라이브에도 사용할 수 있는 충분한 공간이 없으면 디스크 중 일부
를 삭제하고 설치 프로그램을 다시 실행합니다. 삭제할 수 있는 파일에
대하여는 이 설명서의 "디스크 공간 부족" 항목을 참조해 주십시오.

8. CAB 파일 오류 메시지

CAB 파일 오류는 바이러스, 기본 메모리 부족, 원본 디스크의 불량 또는
하드웨어 문제, 특히 CD-ROM 드라이브의 문제등 중 하나 이상이
원인이 되어 흔히 발생할 수 있습니다.

시동 디스크가 있다면 설치를 중단하고 시스템을 종료한 다음
시동 디스크를 사용하여 시스템을 재시작한 후 MS-DOS 명령 프롬프트에서
설치 프로그램을 실행합니다.

손상된 CAB 파일 오류 메시지를 복원하려면 아래의 순서를 따라
설치 파일을 하드 디스크에 복사하고 하드 디스크에서 설치
프로그램을 실행합니다.

1. 하드 디스크에 파일을 저장할 200 MB 정도의 공간과 임시
   디렉터리가 필요합니다.
2. Windows 98 시동 디스크로 부트하고 CD-ROM 드라이버를 로드하는
   옵션을 선택합니다. Windows 98 시동 디스크로 부트 시 CD-ROM
   드라이브를 액세스할 수 없으면 반드시 CD-ROM 드라이브 설치시
   함께 제공된 드라이버를 로드해야 합니다.
3. CD-ROM 드라이브 드라이브를 변경하고 Win98 폴더로 이동합니다.
   'Copy *.* c:\temp' 또는 'Copy *.* d:\temp'를 입력합니다.
4. 임시 폴더로 이동한 후 설치 프로그램을 실행하고 [안전 복구]를
   선택합니다.

기본적으로 MS-DOS에서 Windows 98 을 설치하는게 좋습니다.


9. 메시지 SU0010, SU0012, SU0015, SU0016

설치 중 이들 메시지가 표시되면 그것은 "Windows NT를 실행하는
시스템에서의 Windows 98 설치 나 "OS/2를 실행하는 시스템에서의
Windows 98 설치" 이기 때문입니다.

메시지 SU0011
하드 디스크가 암호로 보호되어 있으면 설치를 성공적으로 완료할
수 없습니다. 우선 암호 보호를 제거해야 합니다. 자세한 내용은
컴퓨터 설명서를 참조해 주십시오.

메시지 SU0013
Windows 98을 설치하려면 시동 드라이브가 반드시 MS-DOS 부트
분할 영역이어야 합니다. 시동 드라이브가 HPFS나 NTFS로 포맷되어
있는 경우, 설치를 실행하기 전 MS-DOS 부트 분할 영역을 작성해야
합니다. MS-DOS 부트 분할 영역 작성에 대한 자세한 내용은 컴퓨터
설명서를 참조해 주십시오.

EZ와 같은 협력업체 분할 소프트웨어를 사용하거나 디스크 관리자가
설치되어 있는 경우에도 이 오류가 발생할 수 있습니다.
이런 경우, 시스템을 다시 시작하고 MS-DOS 프롬프트에서 설치 프로그램을
실행시키면 됩니다.


10. 표준 모드 메시지

아래의 오류 메시지가 나타나면 Config.sys 파일에서 메모리 관리자
(EMM386.exe, QEMM, 386Max)를 제거한 다음 설치 프로그램을 다시 실행해
주십시오.

  Standard Mode: Invalid DPMI return.
  Standard Mode: Fault in MS-DOS Extender.
  Standard Mode: Bad Fault in MS-DOS Extender.
  Standard Mode: Unknown stack in fault dispatcher.
  Standard Mode: Stack Overflow.

참고: 문제가 지속되면 EMM386.EXE을 다시 Config.sys 파일에 추가하고
      대신 옵션을 주어 메모리에서 모든 범위를 제외시킵니다.

        예: device=c:\windows\emm386.exe x=A000-FFFF

위 메시지가 표시되거나 시스템이 설치 중 응답을 멈추면 SmartDrive에서
이중 버퍼링을 설정합니다.
일부의 SCSI 하드 드라이브 또는 ESDI 드라이브에는 이중 버퍼링을
사용해야 할 때도 있습니다.

이중 버퍼링을 설정하려면 Config.sys 파일의 시작 부분에 아래 줄을
추가합니다.

        device=c:\windows\smartdrv.exe /double_buffer+

여기에서 "c:\windows" 는 Windows 디렉터리에 대한 경로입니다.


11. "시스템 파일을 백업할 수 없습니다" 메시지

시스템 파일을 저장하는 중 오류 메시지가 표시되면 압축 디스크의
디스크 공간 부족이 원인일 수 있습니다.
불필요한 파일을 삭제하여 시스템 파일을 저장할 드라이브의
(기본 드라이브는 C) 공간을 확보합니다.

일반적으로 대부분의 시스템에서 Windows 98의 제거 정보 저장에 필요한
디스크 공간은 75 MB 정도입니다.

12. "이 시스템 구성에서는 계속 진행할 수 없습니다" 메시지

이 메시지가 표시되면 오래된 비호환 디스크 분할 영역이 있는
것일 수 있습니다. 설치 프로그램을 실행하기 전 데이터를 백업한
다음 디스크를 Fdisk로 다시 분할해야 합니다.

13. 메시지 SU0167

'Desktop' 이라는 파일이나 폴더가 시스템에 있습니다.
현재 Desktop 폴더의 이름을 다른 이름으로 변경하거나 폴더의 위치를
다른 드라이브로 이동한 후 설치 프로그램을 다시 실행합니다.

14. 메시지 SU0410

파일이 손실 또는 손상되었거나 시스템 메모리가 부족하여 필요한
파일을 열 수 없습니다.

설치를 중단하고 시스템을 종료한 다음 시스템을 재시작한 후 MS-DOS 명령
프롬프트에서 설치 프로그램을 실행합니다.

15. "파일을 복사할 수 없습니다." 메시지

복사할 수 없는 파일을 생략합니다. Windows 95 상에서 Windows 98 을 설치
하거나 Windows 98 1998 이하 버전에서 Windows 98을 설치하는 경우에
나타날 수 있는 메시지 입니다.

이 경우 설치를 중단하고 시스템을 종료한 후 시동 디스크를 사용하여
시스템을 재시작한 후 MS-DOS 명령 프롬프트에서 설치 프로그램을 실행하면
대부분 해결됩니다.

16. SU99xxxx 메시지
SU99는 설치 프로그램에서 해당되는 특정한 오류 메시지가 없는 모든 오류에
공통적으로 나타나는 오류 번호입니다.
이 오류는 많은 경우, 부족한 기본 메모리로 인해 발생합니다.

시스템을 재시작한 후 MS-DOS 명령 프롬프트에서 설치 프로그램을 실행
하면 대부분 해결됩니다.


17. "시스템 하드 디스크 확인에 필요한 기본 메모리가 부족합니다."
    "시스템의 하드 디스크를 확인할 수 없습니다. "

이런 오류가 발생하면 하드 드라이브의 확인을 위하여 디스크 검사를 실행할
수 없습니다. 기본 메모리가 부족하거나 설치 시 실행되는 디스크 검사
프로그램에서 수정할 수 없는 오류가 디스크에 있기 때문입니다.

이 오류를 해결하려면 MS-DOS 모드로 시스템을 다시 시작하여 ScanDisk/all을
실행한 후 설치 프로그램을 다시 실행해 주십시오.
그 후에도 문제가 지속되면 MS-DOS 모드로 시스템을 다시 시작한 후
Scandisk /all /surface 옵션을 주어 실행합니다.
이 옵션을 사용하는 경우 검사에 더 많은 시간이 소요되지만 하드 디스크의
상태를 보다 자세히 확인할 수 있습니다.
이 옵션으로 디스크 검사를 실행한 후에도 설치를 계속 진행할 수 없는 경우
에는 Setup /is를 실행하여 설치 시 디스크 검사를 생략할 수도 있습니다.

그러나, 디스크 검사의 생략은 결코 바람직하지 않으며 디스크에 오류가 없다
는 확신이 서는 경우에만 사용해야 합니다.
디스크에 오류가 남아있는 경우에는 설치가 완료되지 않을 수도 있으며
더 나아가 시스템이 더 이상 시동되지 않을 수도 있기 때문입니다.
