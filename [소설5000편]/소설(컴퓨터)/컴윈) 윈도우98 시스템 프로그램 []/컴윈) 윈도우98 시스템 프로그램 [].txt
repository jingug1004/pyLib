▶윈도우98 시스템 유틸 가지고 놀기◀



수없이 들으셨겠지만 윈도우98에는 수많은 시스템 유틸리티들이 추가되었습니다.
이들은 시작메뉴에 등록이 되는 것도 있고 되지 않는 것들도 있습니다.
이들을 잘 이용하면 윈도우98의 달인이나 적어도 꿀리지는 않는 수준의 유저가
될수 있으실 겁니다.

다음부터 소개되는 내용은 윈도우98내의 시스템 유틸리티들에 대해서 사용자들이
알기쉽게 그들의 작동원리나 사용방법을 나름대로 정리해 본것입니다.

아무쪼록 즐거운 시간 되시길.....왠 웨이터 맨트???



가) 레지스트리 검사기 (SCANREGW.EXE) cf) 도스용:scanreg.exe
  
  이 프로그램은 다들 아시다 시피 레지스트리 검사기입니다.
  레지스트리 검사기는 레지스트리 문제를 찾아 수정하는 시스템 관리 프로그램
입니다. 컴퓨터를 시작할 때마다 레지스트리 검사기는 레지스트리의 불일치 
데이터 구조를 자동으로 검사합니다. 또한 레지스트리 검사기는 레지스트리를 
매일 백업합니다.


레지스트리 검사기가 레지스트리의 심각한 문제를 발견하면 곧바로 백업 사본을 
사용하여 레지스트리를 복원하게 됩니다.


레지스트리 검사기는 컴퓨터를 성공적으로 시작한 레지스트리의 압축 백업을 다섯 
개까지 유지합니다. 이 백업 파일은 윈도우 폴더에 위치한 SYSBCKUP 폴더 안에 
저장됩니다. 


레지스트리 검사기는 이 백업본을 5개 가지 저장해놓고 있기 때문에 상당히
편리합니다. 95때를 생각해 보십시요. 윈도우가 스스로 system.dat와 
user.dat의 백업본을 만들어 놓기는 하지만 하나만 해놓기 때문에 유용하게
쓰일때는 거의 없었습니다.


만일 레지스트리 검사기가 레지스트리 오류를 제대로 잡아내지 못하거나 
작동에 문제가 생겨서 윈도우를 작동시킬수 없는 상황에 빠지더
라도 SYSBCKUP 폴더안의 레지스트리 백업본에서 레지스트리 파일을 추출하여 윈도
우 폴더안에 복사해 주면 수동으로 윈도우를 복구할수 있습니다. 레지스트리 백업본
은 cab형식으로 압축되어 있는 상태이기 때문에 이를 사용하기 위해서는 command 
폴더안의 extract.exe를 사용하여 압축을 풀어주고 (예: extract.exe rb001.cab) 
system.dat 와 user.dat 파일을 윈도우 폴더에 복사해 주면 됩니다.


  또한 설치 프로그램은 컴퓨터 운영 체제를 업그레이드할 때마다 레지스트리 
검사기를 자동으로 실행합니다. Windows 98을 설치할 때 레지스트리 검사기는 
레지스트리 문제 대부분을, 사용자가 미처 알지 못한 문제까지도 수정해 줍니다.

  
  시스템은 레지스트리 구성(사용자 계정 정보, 프로토콜 바인딩, 소프트웨어 프로
그램 설정, 사용자 기본 설정 포함)의 백업 사본을 항상 보존해 둡니다. 현재 레지
스트리에 문제가 발생하면 백업 사본을 사용할 수 있습니다. 컴퓨터를 다시 시작할
때마다 레지스트리 검사기는 레지스트리를 자동으로 검색합니다. 
레지스트리 검사기가 문제를 발견하면 자동으로 레지스트리를 백업 사본으로 
바꿉니다.


  여기서 주목해야 할 점은 vxd 파일과 같이 더 이상 존재하지 않는 파일을 참조
하는 항목이 레지스트리에 포함되어 있으면 레지스트리 검사기로 레지스트리를 수정
할 수 없다는 사실입니다. 이점을 방지하기 위해서는 Regclean 같은 유틸리티를 
이용해서 레지스트리를 자주 청소해 주어야 할 것입니다.


아울러 이 작업은 도스상에서도 하실수 있습니다.
scanreg.exe를 실행하시면 도스용 레지스트리 검사기가 실행
됩니다, command 폴더안에 있습니다.
scanreg /fix 를 사용하시면 레지스트리를 재정렬해줍니다.
가끔씩 해주면 속도향상에 도움이 되겠지요. 왜냐구요? 그야 레지스트리의
크기가 줄어들게 되기 때문이죠. 더불어서 정보들이 차곡차곡 정리되어 있으면
시스템에 에러가 발생할 확률도 현저히 줄어들겠죠.

재정렬 작업은 반드시 도스 상태에서 도스용 레지스트리 검사기를 사용하셔야
합니다. 


나) Dr. Watson (DRWATSON.EXE) 

  Dr. Watson은 시스템 고장이 발생할 때마다 시스템의 스냅 샷을 만드는 진단 
도구입니다. 스냅샷이란 말그대로 문제가 발생했던 때의 윈도우 환경을 사진을 
찍듯이 파일에 기록해 놓는다는 것입니다. 윈도우 3.1때에도 존재했던 프로그램
인데 이번에는 그 성능이 월등히 향상되었습니다. Dr. Watson은 소프트웨어 고장을
알아내고 오류가 있는 소프트웨어를 식별하고 원인에 대한 자세한 설명을 제공합니
다. Dr. Watson은 종종 문제를 진단하고 조치를 제안합니다. 


  흔희들 이 닥터 왓슨의 등장을 대단히 놀라운 것으로 평가합니다. 닥터왓슨은 
윈도우3.1에서도 존재했었는데 95에서는 잠시 사라졌다가 98에서 부활했습니다.
물론 기능은 3.1때의 그것에 비할바가 아닙니다. 하지만 닥터왓슨의
기능은 한계가 있어서 노턴 크래쉬 가드따위의 프로그램들 처럼 에러를 방지해 
주지는 못합니다. 하지만 에러가 발생한 이유를 분석해주기 때문에 여러모로
유용한 프로그램이라고 하겠습니다.


  Dr. Watson은 기본적으로 로드 되지 않습니다. Dr. Watson을 자동으로 
시작하려면 시작프로그램 그룹에 \Windows\DrWatson.exe의 단축 아이콘을 
만들어야 합니다. 
이렇게 해두면 윈도우를 시작할 때마다 닥터왓슨이 자동으로 로드 되어 윈도우의 
에러를 감시하게 됩니다. 
 

다) 시스템 파일 검사기(SFC.EXE)

  시스템 파일 검사기는 컴퓨터를 실행하는 데 필요한 중요한 파일을 기억하고 
있습니다. 시스템 파일을 사용자가 임의로 이동하거나 변경하면 시스템 파일 검사
기는 파일을 복원합니다.

  
다시 말해서 사용자가 중요한 드라이버나 DLL 파일을 삭제해버렸을
경우에 시스템 파일 검사기를 실행시키면 자동으로 이 파일을 복원시켜준다는
이야기입니다. 


시스템 파일 검사기를 사용하면 운영 체제 파일의 무결성을 검증하거나, 손상된 
운영 체제 파일을 복원하거나, 설치 디스크에서 압축 파일(예: 드라이버)의 압축을 
해제할 수 있습니다. 
또한 원본 파일을 복원하기 전에 시스템 파일 검사기로 기존 파일을 백업할 수 
있습니다. 폴더와 파일 확장 명을 기준으로 검색 조건을 사용자 정의할 수도 
있습니다.


  검사를 통해서 작성되는 로그 파일은 Default.sfc라는 바이너리 파일로 저장이 
되며 Sfclog.txt라는 텍스트 파일로도 저장이 되어 자세한 과정을  쉽게 알아볼 
수가 있습니다. 그리고 작성된 로그 파일을 토대로 다음번 검색시 서로 비교하여 
에러를 찾아내게 됩니다.


  SFC.EXE를 실행시키면 프로그램이 가동되는데 여기서 시스템을 검색하기 전에 
몇가지 설정을 해줄수 있습니다. 이 설정 대화상자를 통해서 시스템  파일에 
오류가 있을시에 복원여부를 결정할 수가 있으며 검색기준을 세울수가 있습니다. 
자신이 보기에 검색을 안해도 되는 부분을 제거 해주거나 검색을 필히 해주어야 
되는 곳을 추가해줌으로서 시스템 파일 검사의 효과를 높일수가 있습니다. 이러한 
설정과정이 마무리 되고 나면 곧바로 시스템을 검색하여 문제가 발생한 파일들에 
대해 복구작업을 진행할수 있습니다.

라) Windows Update (WUPDMGR.EXE)

  다들 아시고 계시듯이 윈도우98에서는 Microsoft의 새로운 웹 기반 
리소스 사이트인 Windows Update를 통하여 드라이버와 시스템 파일을 자동으로 
업데이트하거나 최신 정보를 알아 볼 수 있습니다. 

Windows Update에서는 컴퓨터의 장치 드라이버와 시스템 소프트웨어를 검색하고 
그 결과를 웹의 마스터 데이터베이스와 비교한 다음 컴퓨터에 특정 업데이트를 
권장하여 설치합니다.  제거 옵션을 사용하여 이전 장치 드라이버나 시스템 파일로 
되돌릴 수도 있습니다. 

마치 노턴으로 유명한 시만텍社의 프로그램들이 제공하는 
LiveUpdate를 도용한 것 같은 느낌을 주지만 편리한 것은 사실입니다.
WUPDMGR.EXE를 실행시키거나 시작메뉴에 등록되어 있는 'Windows Update'를 
선택해주면 사용하실 수 있습니다. 


 
마) 시스템 정보(MSINFO32.EXE)

  도스 시절의 MSD 의 버전업판으로 보기에는 그 기능이 너무나도 뛰어난
새로운 시스템 정보 유틸리티인 시스템 정보입니다.

  시스템 정보는 시스템 구성 정보를 수집하고 관련 시스템 항목을 표시하기 
위한 메뉴를 제공합니다. 

"마이크로소프트의 기술 지원 담당자가 구성 문제를 
해결하기 위해서는 해당 컴퓨터에 대한 정보를 알아야 합니다" 라고 도움말에는
명시되어 있지만 실제 마소의 기술지원 담당관이 일반 유저들의 시스템의
문제점을 해결해주는것을 기대하기란 어려운 일이죠.

하지만 적어도 시스템 정보를 사용하면 시스템 문제를 해결하는 데 필요한 
데이터를 빨리 찾을 수 있습니다. 

  [시작]을 클릭하고 [프로그램], [보조프로그램], [시스템 도구]를 차례로 
가리킨 다음 [시스템 정보]를 클릭 하여 시스템 정보를 시작할 수 있습니다. 

이 시스템 정보 유틸리티는 크게 두 개의 창으로 나뉘어져 있는데 왼쪽 창은 
트리구조로서 리소스, 구성요소, 소프트웨어 환경의 3가지 항목으로 구성되어 
하드웨어 정보와 소프트웨어에  대한 자세한 정보를 제공합니다. 탐색기와 
마찬가지로 해당항목을 마우스로 클릭해 주면 오른쪽 창에 자세한 정보가 
나타나게 됩니다.
  
  시스템 데이터를 표시하려면 창의 왼쪽 구역에서 더하기 기호(+)를 클릭 하여 
시스템 정보를 확장 표시한 다음 해당 항목을 클릭 합니다. 항목에 따라 기본, 
고급, 이력 시스템 데이터를 표시합니다.  이 응용 프로그램을 사용하면 데이터를
표시할 수 있도록 시스템 데이터를 시스템 정보 파일에 저장할 수 있습니다. 
또한 데이터를 텍스트 파일에 저장하여 컴퓨터에서 팩스로 전송하거나 다른 응용 
프로그램에서 볼 수 있습니다. 데이터 검색이 예상보다 시간이 더 걸리면 사용자는 
검색 과정을 취소할 수도 있습니다. 
  
  또한 시스템 정보유틸리티는 시스템 관리에 사용되는 여러 가지 도구들을 빠르게
사용할수 있도록 프로그램 메뉴 내에 등록시켜 놓았습니다. 메뉴에 등록되어 있는 
프로그램에는 시스템 파일 검사기, 레지스트리 검사기, 자동 생략 드라이버 
에이전트, 닥터 왓슨, 시스템  구성 유틸리티, 디스크 검사, 버전 충돌 관리자가 
있습니다. 
  
  다음은 구성별 설명입니다. 

  리소스 : 충돌/공유, DMA, 그리고 강제로 구성된 하드웨어,  I/O, IRQs, 
메모리에 대한  정보를 얻을수 있는 부분입니다. 충돌/공유 란에서는 시스템의 
장치들이 사용하는 충돌  되었거나 공유되고 있는 각각의 자원들에 대해서 정보를 
제공합니다. DMA(Direct  Memory Access) 및 IRQs(Interrupt Request Query)부분은 
시스템 장치들의 안정적인 작동을 위해서 알아둬야 할 자원 배분에 대한 정보입니다. 

  DMA라는 것은 주변장치와 메모리사이의 직접적인 데이터 전송방식
을 말하는 것입니다. 데이터가 전송될 때 CPU를 거치지 않기 때문에  그만큼 
CPU에 걸리는 부하가 감소됩니다. IRQ 라는것은 특정 하드웨어가 사용되기 위해서 
CPU에 거는 총 16개의 인터럽트  통로를 의미합니다. 
따라서 하나의 장치는 하나의 IRQ를 가져야 하며 중복이 되었을때는 하드웨어의 
충돌로 이어집니다. 따라서 이들을 잘 관리해줘야지만 시스템 자원의 배분에 
관련된 문제를 해결해 나갈수 있습니다.



  구성요소 : 멀티미디어, 디스플레이, 적외선, 입력, 기타, 문제장치, 저장공간
등에 대한 정보를 열람하실수 있습니다. 멀티미디어 란은 시스템 내의 멀티미디어 
환경을 정리하여 한눈에 알아볼수 있도록 데이터를 제공합니다. 시디롬의 간단한 
벤치마크 기능도 포함하고 있어서 자기 시스템에 부착된 시디롬의 성능을 쉽게 
알아볼수 있는 기회를 제공해줍니다. 문제장치 항목은 현재 시스템에 부착된 
하드웨어 중에서 이상이 있는 장치를 위해서 문제 해결을 위한 도움말을 제공하는 
기능을 가지고 있습니다. 


 소프트웨어환경 : 현재 시스템에 로드되어 있는 16bit, 32bit 모듈에 대한 정보와
현재 진행중인 작업에 대한 정보를 얻을수 있습니다. 이 부분에는 현재 로딩되어 
있는 16bit, 32bit 모듈의 버전, 제조업체, 파일의 위치, 파일명, 모듈에 대한 
자료들이 표시되므로 사용자가 모듈에 대해서 알고 싶은 정보들을 쉽게 열람할수 
있습니다.


바) 시스템 구성 유틸리티(MSCONFIG.EXE)

  시스템 구성 유틸리티(Msconfig.exe)는 Microsoft 기술 지원 엔지니어가 
Windows 98 구성 문제를 진단할 때 사용하는 일반적인 문제 해결 단계를 자동화한 
것입니다. 확인란의 선택을 해제하여 시스템 구성 파일을 수정할 수 있으므로 
기존에 메모장이나 시스템 구성 편집기를 사용했기 때문에 발생했던 입력 오류를 
줄일 수 있습니다.
  
  이 프로그램을 사용해서 시스템 파일이나 초기화 파일들, 그 밖의 부팅 관련 
설정들을 조정하실 수 있습니다. 총 6개의 탭으로 구성되어 있고 부팅이 로드 
되는 장치 드라이버와 프로그램 로드를 사용자가 직접 설정하실 수 있습니다. 
그 밖의 config.sys autoexec.bat, win.ini system.ini 등의 초기화 파일을 
기존의 입력방식이 아닌 메뉴방식을 이용하여 보다 쉽게 편집할 수 있게 
도와줍니다. 이 부분은 시중의 레지스트리 편집 관련 프로그램의 컨셉을 마소가 
응용한 것으로 보입니다. 
  
  일반항목에서는 윈도우의 시동시의 환경을 지정하도록  할 수가 있습니다. 
일반적 시작,  분석적 시작, 그리고 선택적 시작의 3가지 부팅 방법을 제공하고 
있습니다. 일반적 시작은 기본적으로 지정되어있는 옵션이고 부팅시 모든 장치의 
제어기들과 프로그램들을 읽어들이게 할 수가 있습니다. 분석적 시작은 장치 
제어기와 프로그램들을 사용자가 부팅시 직접 제어할 수가 있습니다. 
고급 설정부분에서는
              
              ┌ 호환성 모드 디스크 엑세스(도스호환모드) 사용유무
              │ 시스템이 비정상 종료시에도 자동 스캔 안함
              │ 메모리 제한
              │ 빠른 시스템 종료기능 사용안함
              │ 시스템 ROM 중단점 사용안함
              │ 가상 HD IRQ 사용안함
              │ EMM 제외 
              │ UDF 파일 시스템 사용안함
              └ 펜티엄 F0 문제 해결 사용 
               
               등등의 세부적인 설정을 하실수 있습니다.               
               또한 시스템 시작시 로드되는 드라이버의 로드여부를 
               결정할수도 있습니다.
  

 

 
사) 자동 생략 드라이버 에이전트(ASD.EXE)

  개인적으로 한번도 테스트를 해보지 못한 프로그램입니다.
  정말 이런 기능을 제대로만 수행해준다면 상당히 쓸만한 프로그램
이 될것이라고 생각됩니다.

  만약 자신의 하드웨어 구성에 문제가 있을 경우 자동 생략 드라이버 
에이전트를 통해서 이를 제어할 수가 있습니다. 즉 윈도우 시동시 특정 하드웨어
에  대한 드라이버 세팅에 옵션을 줄 수가 있게 됩니다. 그래서 부팅시 문제가 
생긴 하드웨어의  인식을 건너뛰게 함으로서 하드웨어의 충돌로 인한 문제를 밝혀 
낼 수 있도록 도움을 주는 유틸리티입니다. 자신의 하드웨어 세팅에 문제가 없으면 
"ASD(Automatic Skip Driver)에서 오류를 발견하지 못했습니다"라는  대화상자가 
나타납니다.
