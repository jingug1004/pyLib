 IIS 4.0 을 가지고 FTP 사이트를 구축하는 방법을 간단히 설명하겠습니다.
 먼저 설치 순서를 보면

 NT 4.0 설치 -> 서비스팩 3 설치 -> NT 용 인터넷 익스플로러 4.01 설치
 -> SQL 서버 설치 -> 옵션팩 설치.

 이후에 한글판 서비스팩 4 가 나오면 서비스팩을 덮어 씌우시면 됩니다.
 SQL 설치후 SQL 매니저를 통해 작동하는지를 먼저 점검하십시오.

 설치후 관리도구 -> 옵션팩에 보시면 IIS 매니저를 띄울 수 있습니다.
 IIS 는 MMC(Microsoft Management Console) 1.0 로 관리하게 됩니다. 

 마소 세미나 참석으로 구할 수 있는 SQL 7.0 베타 버전을 설치하면
 MMC 1.1 버전을 사용하게 됩니다. MMC 는 NT 5 부터 관리도구를 통합

 하므로 인터페이스에 친숙해지시는게 좋습니다. IIS 4.0 에 들어가면
 콘솔내에 IIS 와 트랜젝션 서버를 보실 수 있습니다. IIS 의 + 버튼을

 눌러 들어가면 디폴트 FTP 서비스와 디폴트 WWW 서비스 등을 보실 수
 있습니다. 디폴트 FTP 서비스의 등록정보(마우스 오른쪽 버튼 클릭

 혹은 메뉴의 '등록정보' 버튼 클릭)으로 들어가셔서 옵션을 조정해
 주시면 됩니다. 물론 고정 IP 를 가지고 있어야 인터넷 서비스를 

 사용할 수 있습니다. 첫번째 탭인 'FTP Site' 에서는 포트지정, 
 접속 사용자수를 조정과 로그인 정보(아래의 등록정보)를 지정할 수

 있습니다. 두번째 탭인 '보안 계정' 에서는 'anonymous' 지정할 수
 있습니다. 세번째 탭인 '메시지'에서는 FTP 서비스를 접속할 때

 메세지를 지정할 수 있습니다. 등록정보의 탭의 몇부분을 고쳐주면
 NT 에서 간단하게 FTP 사이트를 구축할 수 있습니다.
