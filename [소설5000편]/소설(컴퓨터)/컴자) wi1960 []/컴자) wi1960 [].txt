==============================================================

             다중 운영 환경 조성 및 사용하기

==============================================================


1. PC 에서 사용가능한 운영체제의 종류

   PC 에서 사용가능한 운영 체제는 참으로 많다.
   필자 또한 아직 접해보지 못한 잘 알려지지 않은 운영체제가
   수도 없이 많으리라. 그러나 대충 열거해 보면 다음과 같다.

   1) DOS 호환 환경
      MS-DOS, PC-DOS, DR-DOS, Open-DOS

   2) Windows 호환 환경
      Windows 95, Windows 98, Windows NT

   3) Unix 호환 환경
      Linux, SCO-86, Free BSD, Open BSD, ...

   4) 기타
      OS/2, NextStep, Rapsodi for x86, ...

2. 다중 운영을 위한 관점에서의 각 운영체제의 특징

   다음은 다중 운영 환경을 조성하는데 알고 있어야 할 각 운영체제의
   특징에 대해 살펴보자. 가장 많이 쓰이고 있으며 필자가 경험해 본
   운영체제에 대해서만 언급하겠다.

   1) MS-DOS
      M$ 사에서 초기 CP/M 86 을 인수하여 발표한 운영체제로
      현재 독립 운영환경으로써는 6.3 까지 발표되었다.
      이 운영체제의 설치시 특징이라면 전체 디스크 공간을
      파티셔닝하는 유틸리티로 FDISK.EXE 를 제공하는데
      4 개까지의 파티션만을 지원한다.
      또한 여기서 만들어진 부트 로더는 부팅시 자신이 프라이머리
      파티션이며 액티브 파티션이 아니면 커널을 로딩하지 못한다.

   2) Windows 95/98
      역시 M$ 사의 운영체제로 설치시 MBR(Master Boot Recode, 혹은
      Master Boot Sector 라 불리며 일반적인 컴퓨팅 환경에서 IPL:
      Initial Processor Loader, 초기화 프로세서 적재기 - 의 코드를
      담고 있다) 을 자신의 환경에 맞게 수정한다.
      이 부분이 설치시 사용자를 까다롭게 만드는 부분이다.
      또한 MS-DOS 의 특징을 그대로 가지고 있다.

   3) Windows NT
      자체적으로 OS Loader 를 가지며 보통 NT Loader 라 부른다.

   4) OS/2
      IBM 에서 만든 운영체제로 초기에는 M$ 에서 제작하였다.
      독특하게 Boot Manager 라는 모듈을 가지고 있어 다른 운영체제와의
      공존을 쉽게 하도록 배려한 부분이기도 하다.
      특징은 설치시 MBR 에 OBS(OS/2 Boot System)을 기록하고 이 OBS 가
      하나의 파티션에서 부트 매니져를 로딩한다.

   5) Linux
      PC 에서 가장 많이 쓰이는 Unix 운영체제로 자체적으로 Linux Loader
      (LILO) 라는 시동 관리자를 가지고 있다.
      설치시 LILO 를 플로피 및 MBR 혹은 부트 섹터 중 어디에 설치할 지
      선택할 수 있다. 사용자를 위한 배려에 칭찬을 아끼고 싶지 않은
      운영체제이다.

3. 필요한 사항 요약

   1) 각 OS 별 시동 관리자의 특징 요약

      -----------------------------------------------------------
      | OS         |  DOS | Win 95/98 | Win NT |  Linux  | OS/2 |
      -----------------------------------------------------------
      | 시동관리자 |   X  |      X    |    O   |   O     |  O   |
      |    유무    |      |           |        |         |      |
      -----------------------------------------------------------
      | 시동관리자 |   X  |      X    |   BS   | MBR, BS | MBR+ |
      | 상주 위치  |      |           |        | 선택가능|하나의|
      |            |      |           |        |         |파티션|
      -----------------------------------------------------------

    2) 각 OS 설치시 MBR 변경 여부

      ----------------------------------------------------------------
      | OS        |  DOS | Win 95/98 | Win NT | Linux         | OS/2 |
      ----------------------------------------------------------------
      |    MBR    |   X  |      O    |    X   | LILO 를       |  O   |
      | 변경 여부 |      |           |        | MBR 에 설치시 |      |
      ----------------------------------------------------------------

    3) 기타 사항
       위에 열거한 운영체제 외에도 설치시 MBR 변경 여부만 알면 함께 사용이
       가능하다. 또한 OS/2 의 경우 부트 매니져없이도 사용가능하며,
       LILO 없이 Linux 로 부팅할 수도 있다. 또한 통설로 알고 있는 DOS/
       Windows 95/98 은 첫번째 하드에만 설치해서 사용할 수 있다는 것도
       사실은 그렇지 않다.

4. 유용한 도구들

   다음은 필요하거나 유용한 도구들의 개략적인 설명이다.
   필자는 20 가지 정도의 툴들을 사용해 보았으나 필자의 사적인 습관에 의해
   OS 의 공간에 불필요한 파일들을 만들어내는, 예를 들면 V-Communicaions
   사의 System Commander 등의 유틸은 선호하지 않는다.
   물론 그런 도구들이 더 유용한 경우도 있으나 대개의 경우, 그리고 여태까지
   필요하던 그렇지않던 간에 꾸며볼 수 있는 모든 환경을 꾸미는데 그런 도구의
   사용없이 가능했기때문에 더 이상 언급하지 않기로 한다.
   그런 도구들의 사용에 호기심이 간다면 직접 경험해보기 바란다.
   3, 4, 5 번의 비슷한 도구들이다. 이것들은 OS/2 의 부트 매니져와 같은 기능을
   하며 OS/2 부트 매니져와 함께 쓰일 수도 있다.

    1) OS/2 부트 매니져
       꼭 OS/2 를 사용하지 않더라도 부트 매니져는 여러 경우에 유용하다.
       한가지 단점으로는 부트 매니져 자체가 하나의 파티션을 차지한다는
       것이다. 참고로, 대부분의 사용자가 도스 혹은 Windows 95 를 사용하므로
       인해 거의 모든 파티셔닝 툴들이 도스에서 정한대로 디스크당 4개까지의
       파티션 개수에 한계를 가진다. 그런 맥락에서 어쩌면 OS/2 의 부트 매니저
       가 하나의 파티션을 차지하는 것이 커다란 단점일 수도 있다.
       OS/2 의 부트 매니져를 설치하면 MBR 영역에 OBS 라는 부트 매니져 로더가
       설치되고 실제 부트 매니져는 3메가짜리 파티션 하나를 만들어 그곳에서
       운영된다. OS/2 의 부트 매니져를 얻는 방법은 참으로 간단하다.
       각 통신망에 올라있는 OS/2 의 평가버젼 및 베타 혹은 감마 버젼의 설치
       디스크를 다운 받아 필요한 파일들을 디스크에 담으면 된다.
       필요한 디스크는 각각 설치 디스크, 1-2 번 디스크로 총 3장이다.
       하지만 여기서 부트 매니져를 사용하기 위한 파일들만 추려내면 단 한장의
       3.5 인치로 가능하다. 필요한 파일은 대충, 부팅에 관련된 파일, 디폴트
       커맨드 쉘(CMD.EXE), 부트 매니져를 설치하기 위한 FDISK.EXE 이다.
       전체 파일 목록은 따로 쓰지 않겠다. 부팅 디스크를 만들기에 실패한
       독자는 필자에게 의뢰하기 바란다. 잘 만들어진 부팅 디스크의 이미지를
       보내주도록 하겠다(먼저 노력해보라).
       참고로 OS/2 의 최신 버젼은 4.0 인 멀린이다. 그러나 꼭 4.0 의 부트
       매니져일 필요는 없다.

    2) Patition Magic
       FDISK 를 대체하는 더 나은 여러 옵션을 지원하는 파티셔닝 툴이다.
       최신 버젼은 3.04 이며, 이것으로 할 수 있는 것은 참으로 많다.
       특정 파티션을 숨기기, 활성화 하기, 데이터를 잃지 않고 파티션의
       크기를 늘이거나 줄이기, FAT 32 <-> FAT 12 혹은 16 간의 전환,
       특정 파티션을 활성화 하기 등등...
       여기서 우리가 가장 많이 사용하게 될 것은 특정 파티션 숨기기이다
       (Hide 기능). 이유는 앞으로 설명할 것이다.

    3) Master Boot(MR-Boot)
       제작자는 잘 기억나지 않는데 OS/2 부트 매니저류의 프로그램이다.
       Fat32 까지 지원하며(이것의 지원이 별다른 기능은 아니다)
       쉐어웨어로 정식 버젼에서는 6 개, 쉐어웨어 버젼에서는 3 개까지의
       OS 를 지원한다. 부팅시 패스웨드 등의 옵션을 사용가능하다.
       OS/2 의 부트 매니져처럼 파티션을 차지하지 않으며 단지 MBR 만을
       차지한다. 최신 버젼은 2.3 이며 2.3에서 필자는 버그를 발견하였다.
       어떤 문제인지 정확히 모르겠지만 2.2 버젼을 권장한다.
       OSC 의 자료실에 2.1 버젼이 올라와있으며 자체적으로 EFDISK 라는
       파티셔닝 유틸을 제공한다.

    4) OBS 2.08 Beta
       기능은 MR-Boot 와 같으며 IBM 에서 만들었다.
       OS/2 의 부트 매니져의 공개용 버젼으로 알고 있는데, 부트 매니저와는
       다르게 파티션을 차지하지 않으며 MBR 만을 차지한다.

    5) Boot System Loader
       역시 MR-Boot 류의 유틸로 쉐어웨어이다. 지원하는 기능도 MR-Boot 와
       흡사하며 쉐어웨어판에서 특별한 제한도 없다. 또한 OS 선택시 플로피
       도 지원한다. 제작자의 이름을 따서 B...(이름이 잘 기억나지 않는다)
       System Loader 이라 부르기도 한다.

    6) Boot Patition Selector(?)
       정확한 명칭이 기억나지 않는데 2.0 이 최신 버젼이며 NT 의 OS Loader
       에 다른 파티션의 OS 를 쉽게 등록해주는 유틸이다.
       NT 를 디폴트 OS 로 사용하는 사용자에게 유용할 것이다.

    7) 기타
       M$ 에서 자사의 OS 인 Windows 95/98 을 여러 언어 버젼을 설치해서
       선택 사용가능하게 하는 유틸을 발표하였다. System Commander 처럼
       약간은 지저분한 방법을 사용하지만, 하드 디스크가 하나 뿐이거나
       여분의 파티션이 없는 사용자들에겐 유용할 것이다.
       그러나, 2개의 Windows 95/98 을 쓰기 위해서라면 사용하지 말것을
       권장한다. 이때는 그냥 2개의 파티션을 사용하라.
       단지 3개 이상의 Windows 95/98 을 사용할때는 DOS 파티션 하나와
       확장 도스 파티션 하나를 잡고 그것을 논리 드라이브로 나눠서
       각각의 드라이브에 Windows 95/98 을 설치한다면 파티션을 아낄 수
       있다. 이상은 디스크당 파티션이 4개까지로 제한되므로 여러 OS 를
       설치하는데 파티션이 모자라는 상황을 생각해서 설명한 것이므로
       관련없는 사용자는 무시하기 바란다.
       참고로 위에서 열거한 툴들의 대부분은 국내 사설 BBS 에서 구할 수
       있다.

5. 사용 툴의 조합 방법

   위에 열거한 툴들을 기호에 맞게 같이 사용하는 법을 설명한다.
   먼저 알아야 할것은 파티션 매직은 아주 유용하다는 것이다. 거의 필수적인
   툴이라고 생각하기 바란다. 한가지 단점은 각 파티셔닝 유틸마다 디스크
   공간을 다르게 인식하므로 가장 많은 공간을 확보할 수 있는 파티셔닝 툴은
   DOS 의 FDISK 임을 알아두기 바란다. 그래서 먼저 파티션 나누는 법을 설명
   하겠다.

    1) 파티셔닝 방법
       필자의 경우 파티션 매직을 애용하지만 최대의 공간을 사용하기 위해
       최종적으로 도스의 FDISK 를 이용한다.
       파티션 매직, DOS FDISK, OS/2 FDISK, Linux 의 파티션 툴이 확보하는
       공간이 약간씩의 차이가 있다. 한가지 예를 들어 설명하겠다.

       파티션 1 = Windows 95
              2 = Linux
              3 = Linux Swap
              4 = Windows NT

       식으로 설치한다면 먼저 DOS 의 FDISK 로 4개의 파티션을 나눈다.
       파티션 매직으로 두번째 파티션을 지운다.
       두번째 및 세번째 파티션 영역을 Linux 파티션으로 만든다(설치한다)
       그런 파티션 매직에서 다음 1번 파티션을 지우고 4번을 숨긴다.
       DOS 의 FDISK 로 다시 1번 파티션을 만든다.
       같은 방법으로 4번을 지우고 1번을 숨긴후 FDISK 로 4번을 다시 만든다.
       이렇게 하는 이유는 Linux 파티션을 만들때 가능한 모든 영역을 사용한
       후 도스의 FDISK 로 보면 디스크 공간을 100% 사용하지 못하고 3~4 메가
       쯤 남아 있는 것을 보게 될 것이다. 이 공간까지 모두 사용하기 위해서
       위와같이 Linux 파티션이 생성된 이후에 DOS 의 FDISK 로 다시 파티셔닝을
       한 것이다. 다른 OS 의 파티셔닝 유틸을 사용할 때도 위와 같이 응용하기
       바란다.

    2) OS/2 부트 매니저만 사용하는 방법
    3) 3, 4, 5 번 툴만 사용하는 방법
    4) 3, 4, 5 번 툴과 OS/2 부트 매니저를 사용하는 방법
    5) NT 로 기동하여 6 번 툴만 사용하는 방법
    6) 모든 것을 다 사용하는 방법

6. 부트 관리 툴 설치와 OS 설치 실제

   먼저 어떤 OS 를 먼저 설치하느냐의 순서는 아무런 문제가 되지 않는다는 것을
   알아두기 바란다. 앞에서 알아본 바와 같이 MBR 영역을 변경하는 OS 를 설치한
   후라면 MBR 영역에 상주하는 부트 관리 툴을 다시 설치해주기만 하면 된다.

   1) 파티션 나누기
      내가 사용할 OS 가 몇개이며 총 소요되는 파티션이 몇개인지, 그리고
      시스템에 만들 수 있는 파티션이 몇개인지 먼저 점검하기 바란다.
      참고로 OS/2 나 Linux 는 2개의 파티션을 요구한다.
      가능하다면 앞에서 설명한 방법으로 디스크의 모든 공간을 다 사용할 수
      있게 파티셔닝하기 바란다.

   2) 부트 관리 프로그램 설치
      OS/2 의 부트 매니져는 OS/2 부팅 디스크로 부팅하여 FDISK 를 실행하여
      설치할 수 있다. FDISK 실행 후 원하는 디스크의 원하는 부분(처음이나 끝)
      에 설치하면 된다. 아주 쉬운 과정이므로 세세한 설명은 약한다.
      3, 4, 5 번의 툴들도 해당 도움말을 보면 간단히 설치할 수 있다.
      또한 OS/2 부트 매니져는 MBR 에 OBS 를 설치하고 별개의 파티션에 부트
      매니져를 설치하는데 OS/2 부트 매니져 설치 후에 3, 4, 5 번의 툴을
      MBR 에 설치하고 OS/2 의 부트 매니저가 설치된 파티션을 등록해줌으로써
      3, 4, 5 번의 툴로 기동하여 OS/2 부트 매니져를 선택할 수도 있다
      (앞장의 4번의 방법이다.).
      NT 사용자라면 모든것이 끝난 다음 6번 툴을 이용하여 나머지 OS 를 NT
      Loader 에 등록할 수도 있다(모든 툴을 다 사용한 방법인데 이렇게 하면
      처음 부팅시 운영체제를 선택하고 여기서 OS/2 부트 매니져를 선택하면
      다시 OS/2 부트 매니져에서 운영체제를 선택하고, 다시 NT 를 선택했다면
      또 한번 OS 를 선택할 수 있는.. 이색적이긴 하나 너무도 번잡한 방법이다).

   3) DOS 의 설치
      DOS 는 디스크에서 프라이머리 파티션이 하나라야만 설치가 가능하다.
      DOS 의 FDISK 로는 하나의 프라이머리 파티션만 만들 수 있으며
      다른 OS 도 프라이머리 파티션에만 설치가 된다.
      다른 OS 의 파티션은 DOS 가 인식하지 못하므로 문제될 것이 없는데
      여러개의 DOS 호환 파티션을 사용하는 OS 를 설치하고자 한다면(예를 들어
      DOS 6.3 과 Windows 95, Windows 98 Beta 를 동시에 사용하려면
      나머지 2개의 파티션을 파티션 매직으로 숨겨놓고 현재 설치하고자 하는
      파티션을 활성화(Active) 상태로 만들어 준 후에 플로피로 부팅하기 바란다.
      그런 다음 일상적인 방법으로 설치를 하면 그뿐이다.
9
   4) Windows 95/98 설치
      DOS와 마찬가지이다. 쉽게 하기 위해서는 다른 모든 DOS 파티션을 숨기고
      윈도우즈 설치를 하는 것이 좋다. 설치 후에는 MBR 을 변경하므로
      부트 관리 툴을 다시 설치해주어야 한다.

   5) 두번째, 세번째... DOS 혹은 Windows 95/98 설치
      역시 앞서와 같이 나머지 파티션을 숨기고 하면 되며 Windows 의 경우
      설치 후 역시 부트 관리 툴을 재설치 해야한다.

   6) Windows NT 설치
      NT 설치시는 주의해야 할 것이 있다.
      이미 설치되어 있는 DOS/Windows 와 같은 파티션에 설치한다면 상관없지만
      그렇지 않다면 반드시 나머지 파티션을 숨겨주기 바란다.
      다른 DOS 호환 파티션을 숨기지 않으면 NT 는 원하는 파티션에 설치되지만
      NT Loader 는 이미 존재하는 DOS 파티션에 설치되어 깔끔하지 못하게 된다.
      가장 나은 방법은 FAT 로 설치하든 NTFS 로 설치하든간에 설치할 파티션을
      미리 DOS 로 포맷해 두고 나머지 파티션을 숨기고 설치할 파티션을 활성화
      해 놓은 뒤에 설치하는 것이다. NTFS 를 원한다면 설치시에 NTFS 로 변환이
      가능하기 때문이다.

   7) M$ 사의 운영체제 설치시 주의점
      M$ 사의 운영체제는 모두 첫번째 하드 디스크에만 설치가 가능하다.
      따라서 두번째, 세번째 디스크에 설치할때는 편법이 필요한데,
      CMOS 나 SCSI Setup 에서 설치할 디스크 이전에 위치한 모든 디스크를
      Disable 시켜주고 하면 된다.
      예를 들면 IDE 방식의 디스크 2개 중 두번째 디스크의 첫번째 파티션에
      Windows 95 를 설치한다면 CMOS 에서 퍼라이머리 마스트의 디스크 타입을
      None 로 해주고 부팅하면 두번째 디스크가 첫번째 디스크가 되는 것이다.
      그런 상태로 Windows 95 를 설치한 후 다시 CMOS 에서 디스크 정보를 이전
      으로 복원하고 부트 관리 툴에서 해당 파티션을 등록하면 된다.

   8) Linux 설치
      Linux 설치는 한가지만 주의하면 된다. LILO 설치시 MBR 에 설치하지 말고
      (MBR 은 부트 관리 프로그램이 차지하고 있으므로) 부트 섹터에 설치해 주면
      끝이다.

   9) OS/2 설치 / 기타 OS
      OS/2 의 설치 후에 MBR 이 변경되므로 OS/2 의 부트 매니져만을 사용하는게
      아니라면 설치 후 부트 관리 프로그램을 재설치하면 그 뿐이다.
      이상에서 보아온 바와 같이 모든 운영체제가 공통적으로 충돌의 소지가 있는
      다른 파티션을 숨기고 첫번째 디스크에서만 설치가능한 경우 설치할
      디스크보다 순서상으로 앞에 위치한 디스크를 일시적으로 Disable 해 준 다음

      설치하고, 설치가 끝난 후 복구해주면 된다.


7. 기타

   이글이 여러 운영체제 사용에 관한 글이므로 각 운영체제 설치시의 다른
   문제점은 언급하지 않았다. 위에 설명한대로 한다면 큰 어려움이 없을 것이다.
   끝으로 생각나는 것 몇가지를 적어보았다.

   1) DOS FDISK.EXE

      MBR 을 사용하는 부트 관리 툴 제거시 MBR 복구 옵션 :

         FDISK /MBR       - 첫번째 디스크의 MBR 을 복구
         FDISK ID /CMBR   - 디스크 번호가 ID(1,2,3,...) 인 디스크의 MBR 복구

   2) FAT 16 에서 2GB 가 넘는 파티션
      MS-DOS 4.0 에서 클러스터 크기를 키워서 2GB 가 넘는 파티션을 지원하도록
      FAT 16 BIG-DOS 파티션이라는 것을 만들었다. MS-DOS 4.0 의 FDISK 나
      파티션 매직 혹은 Windows NT 에서 만들수 있다.

   3) 디스크 낭비의 해결
      - 파티션을 528메가 이하로 나눈다.
      - FAT 32 를 사용한다.
      - FAT 32 지원 Windows(Windows 95 OSR 2.x / Windows 98 Beta) 에서
        FORMAT 명령의 /Z 옵션을 사용한다.
        FORMAT d: /Z:n   n = 클러스터의 섹터단위 크기
                         1 클러스터 = 512 x n Bytes
      - ZipMagic / Power Folder / Free Space 등의 유틸 사용

   4) DOS 에서 NTFS 인식
      자료실에서 NTFS 라는 주제어로 찾아본다.

   5) NT 에서 FAT 32 인식
      NT 5.0 Beta 외에는 해결책이 없다.

   6) NT 설치 후 드라이브 문자 순서의 문제
      NT 설치 후 제일 먼저 DOS 파티션과 NT 파티션의 드라이브 문자가 NT = C,
      DOS = D, E,... 의 순서로 되어 있지 않다면 어떠한 응용프로그램도 설치하지

      말고 디스크 관리자에서 드라이브 문자를 바꾼다.
      근본적인 해결은 NT 설치 전에 모든 DOS 파티션을 숨긴 후 설치하고 설치가
     끝난 후 Unhide 시킨다.

   7) 3,4,5 번류의 부트 관리 툴을 사용해서 부팅시 활성화 외 파티션 숨기기
      각 OS 설치가 끝나고 실제 부트 관리 프로그램으로 부팅하기 전에 반드시
      여러 도스 파티션 중에 가장 선두에 위치한 것을 Active 상태로 해 놓아야
      한다. 또한 파티션 매직으로 부팅 가능한 모든 파티션이 프라이머리 파티션
      인지 확인해야하며 DOS/Windows 95/98 의 경우 가장 선두에 위치한 DOS
      파티션 외의 모든 부팅가능한 DOS 파티션을 Hide 시킨 상태로 시스템을 부팅
      해야 한다.
